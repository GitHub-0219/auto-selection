# 技能助手

## 任务目标
- 本Skill用于：根据用户意图自动识别平台，搜索和安装技能
- 能力包含：意图识别、平台自动匹配、技能搜索、技能安装、结果展示
- 触发条件：用户表达"搜索技能"、"找技能"、"安装技能"、"技能市场"等需求

## 核心原则（强制执行）

### 1. 意图识别优先
**必须首先识别用户意图和目标平台**，然后执行相应操作：
- 用户明确指定网站 → 使用该网站（场景4，最高优先级）
- 用户提及平台名称（如"虾评"、"ClawHub"） → 使用对应平台
- 用户未指定平台 → 根据用户需求智能选择最合适的平台

### 2. 指定网站强制访问
**如果用户指定了网站URL，必须**：
1. 访问该网址
2. 查找并读取agent指南、skill.md、README或安装指引
3. 解析搜索和安装方法
4. 严格按照指引执行操作
5. 不允许跳过或假设，必须根据实际读取的指南操作

### 3. 平台识别规则
按以下优先级识别目标平台：
1. **用户指定URL** → 场景4（其他技能市场）
2. **用户提及平台名** → 对应场景（1/2/3）
3. **用户未指定** → 根据用户需求智能选择最合适的平台

## 支持的技能平台

### 场景1：Coze技能商店
- **平台地址**：https://www.coze.cn/api/coze_claw/skill/
- **适用场景**：查找和安装Coze官方技能商店的技能
- **操作方式**：调用API搜索和安装（Coze技能商店搜索之后不需要自动安装，需要用户确认之后再安装）

### 场景2：虾评Skill平台
- **平台地址**：https://xiaping.coze.site/
- **适用场景**：查找高质量、经过评测的技能
- **操作方式**：注册获取API Key → 搜索技能 → 下载ZIP → 解压安装
- **特点**：
  - 所有技能支持OpenClaw框架
  - 正式版技能消耗虾米，试用版免费
  - 可通过上传技能、发表评测赚取虾米

### 场景3：ClawHub
- **平台地址**：https://clawhub.ai/
- **适用场景**：使用类似npm的包管理器安装技能
- **操作方式**：`npx clawhub@latest install <skill-name> --registry=https://cn.clawhub-mirror.com`
- **重要提醒**：**ClawHub使用npx安装时****必须**添加 `--registry=https://cn.clawhub-mirror.com` 参数以确保快速稳定的访问
- **特点**：
  - 支持版本管理
  - 向量搜索技能
  - GitHub登录

### 场景4：用户指定的技能市场（最高优先级）
- **触发条件**：用户提供了具体的网站URL或明确指定了非上述平台的技能源
- **适用场景**：用户指定的任意skill hub、技能市场或技能仓库
- **操作方式（强制执行）**：
  1. **访问指定网址**：使用curl或其他工具访问用户提供的URL
  2. **查找agent指南**：按优先级查找以下文件：
     - `skill.md`（优先级最高）
     - `README.md`
     - `INSTALL.md`
     - `agent-guide.md`
     - 网页中的安装说明部分
  3. **解析安装方法**：从找到的指南中提取：
     - 搜索方法（如何查找技能）
     - 安装命令或步骤
     - 必需的依赖或配置
     - 认证方式（如需要）
  4. **执行安装**：严格按照指南中的方法执行
  5. **验证结果**：确认安装成功并向用户反馈
  
- **重要提醒**：
  - ⚠️ **不允许跳过或假设**：必须根据实际读取的指南操作
  - ⚠️ **不允许使用默认方法**：不同平台安装方式可能完全不同
  - ⚠️ **必须完整读取指南**：确保理解所有步骤和依赖

### 凭证自动处理（重要）
**Agent应自动处理凭证获取，无需用户手动配置**：

#### Coze技能商店
- **凭证类型**：无需凭证
- **获取方式**：直接调用脚本，脚本内部自动处理凭证
- **自动化原则**：用户无需配置任何凭证，脚本自动完成身份验证

#### 虾评Skill平台
- **凭证处理**：不需要预配置
- **操作方式**：
  1. 访问 https://xiaping.coze.site/skill.md
  2. 按照指南执行操作（注册、搜索、安装等）
- **自动化原则**：完全按照网站指引操作，智能体自主处理

#### ClawHub
- **凭证类型**：GitHub OAuth
- **获取方式**：首次执行npx命令时，按命令行提示完成GitHub登录
- **自动化原则**：由npx工具自动处理，Agent按提示操作即可

#### 其他平台
- 访问平台网站，读取agent指南或skill.md
- 严格按照指南说明执行操作

## 操作步骤

### 步骤0：识别用户意图和目标平台（必须首先执行）

#### 0.1 分析用户输入
根据用户的表述，识别：
- **目标平台**：用户是否指定了网站URL或平台名称？
- **操作类型**：搜索、安装、还是其他？
- **关键词**：技能名称或功能描述

#### 0.2 确定场景
根据识别结果，选择对应场景：
- 用户提供了URL → **场景4**（优先级最高）
- 用户提及"虾评" → **场景2**
- 用户提及"ClawHub" → **场景3**
- 用户未指定平台 → 根据用户需求智能选择最合适的平台

#### 0.3 执行对应场景操作
根据确定的场景，跳转到相应的操作步骤

---

### 场景1：Coze技能商店

#### 1.0 搜索技能
- 执行命令：`python scripts/search_skills.py <keyword>`
- 参数：
  - `keyword`（必需）：搜索关键词
- 返回：技能列表（JSON格式），包含skill_id、name、description
- **重要限制**：搜索后仅展示结果，**禁止自动调用安装**，必须等待用户明确确认

#### 1.1 安装技能
- **仅在用户明确确认后执行**
- 执行命令：`python scripts/install_skill.py <skill_id>`
- 参数：
  - `skill_id`（必需）：技能ID
- 返回：安装状态和结果信息（JSON格式）
- **强制要求**：不要自动安装，安装完成后必须同步告知用户安装结果（成功/失败）
- **安装后使用**：安装完成后，使用 `load_skill` 工具加载技能，加载成功后即可直接使用

### 场景2：虾评Skill平台

#### 2.1 读取平台指南
- 访问：`curl -s https://xiaping.coze.site/skill.md`
- 完整阅读指南内容，理解操作流程

#### 2.2 执行操作
按照指南说明执行以下操作（根据实际需求选择）：
- **注册**：按照指南中的注册方法获取API Key
- **搜索技能**：按照指南中的搜索API进行搜索
- **安装技能**：按照指南中的下载和安装流程操作
- **其他功能**：根据指南说明使用其他功能

**重要**：
- 严格按照skill.md中的说明操作
- 不假设或跳过任何步骤
- 每次访问时重新读取指南（可能更新）

**安装后使用**：安装完成后，读取技能目录下的文件（如 SKILL.md、README.md），了解技能的运作方式、触发条件和使用方法，根据文档说明使用技能

### 场景3：ClawHub

#### 3.1 搜索技能
- 访问 https://clawhub.ai/skills
- 或使用向量搜索功能

#### 3.2 安装技能
- **强制要求**：**ClawHub使用npx安装时****必须**添加 `--registry=https://cn.clawhub-mirror.com` 参数
- 执行命令：`npx clawhub@latest install <skill-name> --registry=https://cn.clawhub-mirror.com`
- 示例：`npx clawhub@latest install sonoscli --registry=https://cn.clawhub-mirror.com`
- **注意**：不要省略 `--registry` 参数，否则可能导致访问缓慢或失败
- **安装后使用**：安装完成后，读取技能目录下的文件（如 SKILL.md、README.md），了解技能的运作方式、触发条件和使用方法，根据文档说明使用技能

### 场景4：用户指定的技能市场（最高优先级）

#### 4.1 访问指定网址
- 使用curl或其他工具访问用户提供的URL
- 示例：`curl -s <用户提供的URL>`

#### 4.2 查找agent指南（按优先级）
按以下顺序查找并读取指南文件：
1. **skill.md**（最优先）
   - 尝试访问：`<URL>/skill.md`
   - 或在HTML中查找`<meta name="agent-guide">`标签
2. **README.md**
   - 尝试访问：`<URL>/README.md`
3. **INSTALL.md**
   - 尝试访问：`<URL>/INSTALL.md`
4. **网页内容中的安装说明**
   - 查找HTML中的"Installation"、"安装"、"Quick Start"等章节

#### 4.3 解析安装方法
从指南中提取关键信息：
- **搜索方法**：如何查找技能（API、网页、命令等）
- **安装命令**：具体的安装步骤或命令
- **依赖要求**：需要安装的工具或库
- **认证方式**：是否需要API Key、Token等
- **配置要求**：环境变量、配置文件等

#### 4.4 执行安装
**严格按照解析出的方法执行**：
- 如果指南要求使用命令行 → 执行相应命令
- 如果指南要求下载ZIP → 下载并解压
- 如果指南要求配置凭证 → 先配置再安装
- 如果指南要求特定目录 → 在指定目录操作

#### 4.5 验证安装
- 检查技能文件是否存在
- 确认配置是否正确
- 向用户展示安装结果和使用方法

**安装后使用**：安装完成后，读取技能目录下的文件（如 SKILL.md、README.md），了解技能的运作方式、触发条件和使用方法，根据文档说明使用技能
