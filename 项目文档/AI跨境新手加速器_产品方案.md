## 一、产品PRD

### 1.1 产品定位

#### 产品名称建议

| 方案 | 名称 | 特点 |
|-----|------|------|
| 方案一 | **EasyTik** | 简单易用，TikTok品牌关联强 |
| 方案二 | **跨境易站** | 接地气，突出跨境电商痛点 |
| 方案三 | **新手出海** | 直击目标用户，情感共鸣强 |
| **推荐** | **EasyTikAI** | AI+平台+简单，国际化风格 |

**最终建议**：采用 **EasyTikAI** 作为产品名称，同时申请 `easytikai.com`、`easytika.com` 等域名。

#### 核心价值主张

> **"30分钟，从跨境小白到TikTok Shop卖家"**

- **效率提升**：将传统需要1-2周的学习选品和内容制作周期，压缩到30分钟
- **门槛降低**：无需专业背景，AI辅助完成选品分析、Listing撰写、视频脚本生成
- **风险可控**：基于数据选品，降低试错成本，提高成功率

##### 功能1：AI智能选品助手

| 项目 | 内容 |
|-----|------|
| **功能名称** | AI智能选品助手 |
| **优先级** | **P0**（核心功能，上线必备） |
| **解决痛点** | 不知道卖什么、不会分析市场数据、选品全靠直觉 |

**功能描述**：
用户输入想做的品类或关键词，AI基于TikTok Shop东南亚各国（印尼、泰国、越南、菲律宾、马来）的市场数据，分析：
- 市场需求量（销量趋势）
- 竞争程度（同类商品数量、平均价格）
- 利润率预估
- 建议定价区间
- 风险提示（是否需要认证、是否有专利风险）

**交互流程**：

```
1. 用户进入「选品助手」页面
2. 选择目标市场（可多选：印尼/泰国/越南/菲律宾/马来）
3. 输入品类关键词（如"厨房用品"）或上传图片识别品类
4. 设置筛选条件：
   - 价格区间
   - 月销量门槛
   - 竞争程度偏好（蓝海/红海）
5. 点击「开始分析」，AI返回选品报告
6. 报告包含：
   - 10-20个推荐商品
   - 每个商品的市场数据图表
   - 优劣势分析
   - 入局建议
7. 用户可收藏商品、查看详细数据、或直接生成Listing
```

##### 功能2：爆品分析工具

| 项目 | 内容 |
|-----|------|
| **功能名称** | 爆品分析工具 |
| **优先级** | **P0**（核心功能，上线必备） |
| **解决痛点** | 不知道什么品在卖、无法复制爆品逻辑 |

**功能描述**：
展示TikTok Shop东南亚各国当前的爆款商品排行榜，支持按类目、按国家、按时间段筛选。用户可以：
- 查看爆品的详细数据（销量、价格、评价）
- 分析爆品的内容策略（视频风格、主播类型）
- 一键参考爆品生成自己的Listing/视频脚本

**交互流程**：

```
1. 用户进入「爆品榜单」页面
2. 选择筛选条件：
   - 国家（印尼/泰国/越南/菲律宾/马来）
   - 类目（美妆/家居/3C/服饰等）
   - 时间范围（今日/本周/本月）
   - 排序方式（销量/销售额/增长率）
3. 系统展示爆品列表（卡片形式）
4. 点击任意商品，查看详情：
   - 基础信息：标题、价格、店铺
   - 销售数据：30天销量趋势图
   - 内容分析：爆款视频截图、达人带货情况
   - 用户评价：高频关键词、痛点需求
5. 用户可：
   - 「相似推荐」：AI推荐相似但竞争更小的品
   - 「参考此品」：生成自己的Listing
   - 「视频灵感」：生成类似风格的视频脚本
```

##### 功能3：Listing一键生成

| 项目 | 内容 |
|-----|------|
| **功能名称** | Listing一键生成 |
| **优先级** | **P0**（核心功能，上线必备） |
| **解决痛点** | 英文/小语种差、不会写高转化标题和描述 |

**功能描述**：
用户上传商品图片或输入商品基本信息，AI自动生成：
- 多语言标题（英文+目标市场本地语言）
- 五点描述（Bullet Points）
- 长描述（详情页文案）
- 关键词标签（Tags）
- 定价建议

支持多平台格式：TikTok Shop、Shopee、Lazada

**交互流程**：

```
1. 用户进入「Listing生成」页面
2. 选择目标平台和国家
3. 输入商品信息（至少一项）：
   - 商品图片（最多9张）
   - 商品名称（中文即可）
   - 商品特点（可多选、可手动输入）
   - 目标价格
4. 点击「AI生成」
5. 系统展示生成结果：
   - 标题（提供3个版本供选择）
   - 描述文案（可直接使用或编辑）
   - Tags（可复制）
   - 定价建议
6. 用户可：
   - 「一键复制」：复制所有内容
   - 「重新生成」：调整风格后重新生成
   - 「保存草稿」：保存到我的商品库
   - 「导出」：下载为Excel/CSV
```

##### 功能4：视频脚本生成

| 项目 | 内容 |
|-----|------|
| **功能名称** | 视频脚本生成 |
| **优先级** | **P1**（重要功能，第二阶段上线） |
| **解决痛点** | 不会拍视频、不知道拍什么、内容没流量 |

**功能描述**：
基于商品信息和目标市场，AI生成多种风格的短视频脚本：
- 种草安利型（适合普通用户）
- 开箱测评型（适合专业内容）
- 对比展示型（适合工厂/贸易商）
- 情景剧型（适合有剧情的产品）

**交互流程**：

```
1. 用户进入「视频脚本」页面
2. 输入基础信息：
   - 商品名称/图片
   - 目标市场
   - 视频时长（15秒/30秒/60秒）
   - 内容风格（从预设中选择）
3. 点击「生成脚本」
4. 系统返回：
   - 分镜头脚本（包含时长、画面、台词、BGM建议）
   - 拍摄要点提示
   - 热门背景音乐推荐
   - 封面标题建议
5. 用户可：
   - 「查看示例视频」：参考同类爆款视频
   - 「重新生成」：换一种风格
   - 「保存收藏」：保存到素材库
```

##### 功能5：新手入门课程

| 项目 | 内容 |
|-----|------|
| **功能名称** | 新手入门课程 |
| **优先级** | **P1**（重要功能，第二阶段上线） |
| **解决痛点** | TikTok Shop规则复杂、新手学习路径不清晰 |

**功能描述**：
系统化的TikTok Shop东南亚从0到1教程，包含：
- 基础认知篇：TikTok Shop是什么、东南亚市场概况
- 入驻流程篇：各国店铺开通、资质要求、注意事项
- 选品方法论：选品思路、数据分析工具使用
- 内容创作篇：短视频制作、直播带货基础
- 物流履约篇：发货流程、海外仓选择
- 政策规则篇：平台规则、违规避坑

**交互流程**：

```
1. 用户进入「学习中心」
2. 选择学习路径：
   - 「新手快速入门」：3天速成课
   - 「选品专项」：深入学习选品
   - 「内容创作」：短视频+直播
   - 「全流程精通」：系统学习
3. 观看课程视频（支持倍速、离线下载）
4. 完成课后测试
5. 获得学习进度和结业证书
6. 可参与「学习社群」与其他学员交流
```

### 1.4 技术选型建议

#### 1.4.1 前端框架

| 框架 | 推荐度 | 理由 |
|-----|-------|------|
| **Vue3 + Vite** | ⭐⭐⭐⭐⭐ | 创始人技术背景Vue优先，开发效率高，社区生态丰富 |
| React | ⭐⭐⭐⭐ | 功能强大，但学习曲线稍陡 |
| Next.js | ⭐⭐⭐ | SSR能力强，但对1万验证阶段略重 |

**推荐配置**：
- **UI框架**：Element Plus（PC端）/ Vant（移动端）
- **状态管理**：Pinia
- **HTTP库**：Axios
- **图表**：ECharts
- **构建工具**：Vite

#### 1.4.2 后端框架

| 框架 | 推荐度 | 理由 |
|-----|-------|------|
| **NestJS** | ⭐⭐⭐⭐⭐ | TypeScript优先，企业级架构，团队扩展友好 |
| Express/Koa | ⭐⭐⭐⭐ | 轻量灵活，适合快速MVP |
| Spring Boot | ⭐⭐⭐ | Java系首选，但启动慢 |

**推荐配置**：
- **语言**：TypeScript（前后端统一）
- **ORM**：Prisma（类型安全，开发体验好）
- **认证**：JWT + Refresh Token
- **定时任务**：Bull（Redis队列）
- **日志**：Winston + ELK

#### 1.4.3 数据库选型

| 数据库 | 用途 | 推荐度 | 理由 |
|-------|------|-------|------|
| **PostgreSQL** | 主业务数据 | ⭐⭐⭐⭐⭐ | 关系型可靠，JSON支持好，JSONB适合灵活字段 |
| **MongoDB** | 非结构化数据 | ⭐⭐⭐⭐ | 商品详情、用户内容等半结构化数据 |
| **Redis** | 缓存/会话 | ⭐⭐⭐⭐⭐ | 高性能缓存、限流、Session存储 |
| **Elasticsearch** | 搜索功能 | ⭐⭐⭐ | 商品搜索、爆品检索 |

#### 1.4.4 AI接口选型

| AI服务商 | 推荐度 | 适用场景 | 成本 | 说明 |
|---------|-------|---------|-----|------|
| **硅基流动 (SiliconFlow)** | ⭐⭐⭐⭐⭐ | 全场景 | 低 | 国产平替，API兼容OpenAI，价格低，支持多种模型 |
| **智谱AI (GLM-4)** | ⭐⭐⭐⭐ | 中文场景 | 中 | 中文理解好，稳定性高 |
| **OpenAI (GPT-4o)** | ⭐⭐⭐⭐ | 高质量生成 | 高 | 效果最好，但成本高+网络问题 |
| **Claude (Anthropic)** | ⭐⭐⭐ | 长文本/分析 | 高 | 长上下文优秀，适合分析 |
| **通义千问/Qwen** | ⭐⭐⭐⭐ | 中文场景 | 中 | 阿里云生态集成好 |

**推荐策略**：
- **MVP阶段**：以硅基流动为主（成本低），智谱AI作为中文备选
- **用户生成内容**：调用硅基流动的GPT-4o-mini
- **选品分析报告**：调用智谱GLM-4（中文理解更好）
- **未来扩展**：预留多模型切换能力

#### 1.4.5 第三方服务接入

| **OSS对象存储** | 阿里云OSS | 存储图片/文件 | 腾讯云COS |
