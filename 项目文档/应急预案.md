## 二、常见故障场景及处理

### 2.1 服务宕机

#### 场景：网站无法访问

**快速诊断：**
```bash
# 1. 检查PM2进程状态
pm2 list
pm2 logs auto-selection --lines 50

# 2. 检查Nginx状态
systemctl status nginx
nginx -t

# 3. 检查端口监听
netstat -tlnp | grep -E '80|443|3000'

# 4. 检查系统资源
htop
df -h
free -m

# 5. 检查网络连通性
curl -I https://www.autoselect.com
curl -I http://127.0.0.1:3000
```

**处理流程：**

| 步骤 | 操作 | 命令 |
|------|------|------|
| 1 | 重启PM2进程 | `pm2 restart auto-selection` |
| 2 | 如仍失败，检查日志 | `pm2 logs auto-selection --err` |
| 3 | 重启Nginx | `systemctl restart nginx` |
| 4 | 如端口被占用，杀掉进程 | `kill -9 <pid>` |
| 5 | 确认服务恢复 | `curl -I https://www.autoselect.com/health` |

**回滚脚本：**
```bash
#!/bin/bash
# rollback.sh - 紧急回滚脚本

echo "开始回滚操作..."

# 1. 停止当前服务
pm2 stop auto-selection

# 2. 切换到上一个稳定版本
cd /var/www/auto-selection
git checkout v0.9.0

# 3. 重新安装依赖
npm install --production

# 4. 重启服务
pm2 start ecosystem.config.js --env production

# 5. 验证
sleep 5
curl -I https://www.autoselect.com/health

echo "回滚完成！"
```

### 2.2 数据库故障

#### 场景：数据库连接失败

**快速诊断：**
```bash
# 1. 检查MySQL状态
systemctl status mysql
# 或
systemctl status mariadb

# 2. 检查数据库连接
mysql -u root -p -e "SELECT 1"

# 3. 检查连接数
mysql -u root -p -e "SHOW STATUS LIKE 'Threads_connected'"

# 4. 检查最大连接数
mysql -u root -p -e "SHOW VARIABLES LIKE 'max_connections'"
```

**处理流程：**

| 步骤 | 操作 | 说明 |
|------|------|------|
| 1 | 重启MySQL | `systemctl restart mysql` |
| 2 | 如无法恢复，检查磁盘空间 | `df -h /var` |
| 3 | 清理临时文件 | 释放空间 |
| 4 | 从备份恢复 | 恢复最近备份 |
| 5 | 验证连接 | 重试连接测试 |

### 2.3 内存溢出

#### 场景：应用内存持续增长或突然爆满

**快速诊断：**
```bash
# 1. 查看PM2内存使用
pm2 list
pm2 monit

# 2. 查看详细内存信息
pm2 show auto-selection

# 3. 查看系统内存
free -m

# 4. 查看内存泄漏日志
pm2 logs auto-selection --lines 100 | grep -i memory
```

**处理流程：**

| 步骤 | 操作 | 命令 |
|------|------|------|
| 1 | 设置内存阈值 | `pm2 restart auto-selection` |
| 2 | 查看是否配置了max_memory_restart | 检查ecosystem.config.js |
| 3 | 开启详细日志 | `pm2 start auto-selection --log /var/log/pm2/debug.log --verbose` |
| 4 | 联系开发分析 | 导出heapdump |
| 5 | 紧急扩容 | 升级服务器配置 |

### 2.4 SSL证书问题

#### 场景：HTTPS无法访问或证书过期

**快速诊断：**
```bash
# 1. 检查证书状态
sudo certbot certificates

# 2. 检查证书有效期
openssl s_client -connect autoselect.com:443 -servername autoselect.com 2>/dev/null | openssl x509 -noout -dates

# 3. 检查Nginx SSL配置
nginx -t
cat /etc/nginx/conf.d/autoselect.conf
```

**处理流程：**

| 步骤 | 操作 | 命令 |
|------|------|------|
| 1 | 证书快过期 | `sudo certbot renew` |
| 2 | 证书已过期 | `sudo certbot renew --force-renewal` |
| 3 | 重载Nginx | `sudo systemctl reload nginx` |
| 4 | 验证 | `curl -I https://autoselect.com` |

**临时解决方案（证书完全不可用时）：**
```nginx
# 临时允许HTTP访问
server {
    listen 80;
    listen 443;
    server_name autoselect.com;
    # 临时跳过SSL验证（仅紧急情况）
}
```

### 2.7 DDoS攻击

#### 场景：服务器遭受DDoS攻击

**快速诊断：**
```bash
# 1. 检查CPU和网络
htop
sar -n DEV 1 10

# 2. 查看连接数
netstat -an | grep ESTABLISHED | wc -l
netstat -an | grep SYN_RECV | wc -l

# 3. 查看访问日志
tail -n 1000 /var/log/nginx/access.log | awk '{print $1}' | sort | uniq -c | sort -rn | head -20
```

**处理流程：**

| 步骤 | 操作 | 工具/命令 |
|------|------|----------|
| 1 | 启用CC防护 | 阿里云安全组 |
| 2 | 配置限流 | Nginx limit_req |
| 3 | 封禁IP | `iptables -I INPUT -s <ip> -j DROP` |
| 4 | 开启高防 | 阿里云DDoS高防 |
| 5 | 联系ISP | 必要时协助清洗 |

**紧急限流配置：**
```nginx
# Nginx限流配置
limit_req_zone $binary_remote_addr zone=one:10m rate=10r/s;
limit_req zone=one burst=20 nodelay;
```

## 三、回滚操作指南

### 3.1 完整回滚流程

```bash
#!/bin/bash
# emergency-rollback.sh

set -e

echo "=========================================="
echo "紧急回滚脚本"
echo "执行时间: $(date)"
echo "=========================================="

# 1. 创建回滚点
BACKUP_DIR="/var/www/backup/$(date +%Y%m%d_%H%M%S)"
mkdir -p $BACKUP_DIR

# 2. 备份当前代码
cp -r /var/www/auto-selection $BACKUP_DIR/code

# 3. 备份当前数据库
mysqldump -u root -p auto_selection > $BACKUP_DIR/db.sql

# 4. 备份配置
cp /var/www/auto-selection/.env.production $BACKUP_DIR/

# 5. 切换到上一个版本
cd /var/www/auto-selection
git checkout v0.9.0

# 6. 重新安装依赖
npm install --production

# 7. 重启服务
pm2 restart auto-selection

# 8. 验证
sleep 10
if curl -sf https://www.autoselect.com/health > /dev/null; then
    echo "✓ 回滚成功！服务已恢复"
else
    echo "✗ 回滚后服务验证失败，请手动检查！"
fi

echo "=========================================="
echo "备份文件位置: $BACKUP_DIR"
echo "=========================================="
```

### 3.2 数据库回滚

```bash
#!/bin/bash
# db-rollback.sh

# 从备份恢复数据库
mysql -u root -p auto_selection < /var/www/backup/YYYYMMDD_HHMMSS/db.sql

# 验证
mysql -u root -p -e "SELECT COUNT(*) FROM users;" auto_selection
```

### 3.3 回滚决策矩阵

| 故障类型 | 影响范围 | 回滚必要性 | 回滚速度 |
|----------|----------|------------|----------|
| 核心功能崩溃 | 全站用户 | 必须回滚 | < 30分钟 |
| 数据丢失 | 部分用户 | 必须回滚 | < 1小时 |
| 安全漏洞 | 全站风险 | 必须回滚 | < 1小时 |
| 性能下降 | 部分用户 | 视情况 | 可延迟 |
| 页面样式问题 | 可忽略 | 不需要 | - |
| 第三方接口异常 | 依赖功能 | 不需要 | - |
