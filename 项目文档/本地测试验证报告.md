## 四、启动脚本验证

### 4.1 脚本路径
`./项目/start-local.sh`

### 4.2 启动流程

```
[1/6] 检查 Node.js 环境
    ↓
[2/6] 检查端口占用 (3000/3001)
    ↓
[3/6] 安装后端依赖
    ↓
[4/6] 安装前端依赖
    ↓
[5/6] 启动后端Mock服务 (端口3001)
    ↓
[6/6] 启动前端服务 (端口3000)
```

### 4.3 关键配置

| 配置项 | 值 |
|--------|-----|
| 后端端口 | 3001 |
| 前端端口 | 3000 |
| 后端入口 | `./项目/backend/mock-server.js` |
| 前端命令 | `npm run dev` |
| 日志目录 | `./项目/logs/` |

### 4.4 健康检查

脚本包含健康检查机制：
- 后端：检查 `http://localhost:3001/api/health` (10秒超时)
- 前端：检查 `http://localhost:3000` (15秒超时)

## 六、启动步骤

### 6.1 快速启动（Mac/Linux）

```bash
cd ./项目
chmod +x start-local.sh
./start-local.sh
```

### 6.2 手动启动

```bash
# 终端1：启动后端
cd ./项目/backend
npm install
node mock-server.js

# 终端2：启动前端
cd ./项目/frontend
npm install
npm run dev
```

### 6.3 Windows启动

```powershell
cd .\项目
.\start-local.sh
```

或使用Git Bash执行。
