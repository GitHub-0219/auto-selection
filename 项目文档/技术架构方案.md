## 一、技术选型

### 1.1 前端技术栈

| 层级 | 推荐方案 | 备选方案 | 选型理由 |
|------|----------|----------|----------|
| **框架** | Next.js 14 (App Router) | Nuxt 3 | SSR/SSG支持好，SEO友好，Vercel免费额度充足，开发体验优秀 |
| **UI组件库** | shadcn/ui + Tailwind CSS | Ant Design | 组件可控、样式灵活、包体积小 |
| **状态管理** | Zustand | Redux Toolkit | 轻量、TypeScript友好、学习成本低 |
| **HTTP客户端** | Axios + React Query | SWR | 请求缓存、自动重试、适合SaaS场景 |
| **图表/可视化** | Recharts | Tremor | 免费、轻量、与React生态集成好 |
| **表单处理** | React Hook Form + Zod | - | 性能好、类型安全、验证强大 |

### 1.2 后端技术栈

| 层级 | 推荐方案 | 备选方案 | 选型理由 |
|------|----------|----------|----------|
| **运行时** | Node.js 20 LTS | Bun | NPM生态最丰富，维护成本低 |
| **框架** | NestJS | Express + 中间件 | 模块化、装饰器写法、团队协作友好 |
| **API风格** | RESTful API | tRPC | 简单直观，第三方集成方便（如TikTok API） |
| **ORM** | Prisma | TypeORM | 类型安全、迁移方便、DevTools可视化 |

### 1.3 数据库选型

| 类型 | 推荐方案 | 备选方案 | 选型理由 |
|------|----------|----------|----------|
| **主数据库** | PostgreSQL (Supabase/Turso) | MySQL (PlanetScale) | Supabase免费额度1GB、支持向量、实时订阅、PG生态成熟 |
| **缓存层** | Redis (Upstash) | - | Upstash免费额度3GB、Serverless计费、兼容Redis协议 |
| **搜索** | Algolia/Typesense | - | Typesense开源可自部署，Algolia有免费配额 |

**验证阶段推荐组合**：Supabase（PostgreSQL + Auth + Storage）一站式解决

### 1.4 AI/大模型接口

| 服务 | 推荐方案 | 备型方案 | 选型理由 |
|------|----------|----------|----------|
| **主力模型** | Kimi API (月之暗面) | 硅基流动 | 中文优化好、成本低、效果稳定 |
| **低成本方案** | 硅基流动 API | DeepSeek API | 性价比高、Llama/国产模型丰富 |
| **图片生成** | Midjourney API | Stable Diffusion API | 质量高、电商场景适配 |

**成本参考**（验证阶段）：
- Kimi：¥15-30/百万tokens
- 硅基流动：¥1-5/百万tokens（部分模型免费）

### 2.2 分层说明

#### 前端层 (Frontend)
- **Next.js App Router**：服务端渲染 + 静态生成
- **Vercel部署**：全球CDN、免费SSL、自动预览部署
- **特性**：PWA支持、响应式设计、多语言（英语/泰语/越南语/印尼语）

#### API层 (API Gateway)
- **认证**：JWT Token + Refresh Token
- **限流**：每用户/每IP多级限流
- **监控**：请求日志、错误追踪（Sentry免费版）

#### 业务层 (Business Logic)
- **NestJS Modules**：用户、商品、订单、AI服务、支付等模块
- **微服务预备**：模块化设计，后续可拆分

#### 数据层 (Data Layer)
- **Supabase**：数据库 + Auth + Storage + Edge Functions
- **Upstash Redis**：会话缓存、API限流计数器、排行榜
- **数据安全**：敏感字段加密传输，数据库行级安全策略(RLS)

### 4.2 核心表结构

#### 用户表 (users)

```sql
CREATE TABLE users (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    email VARCHAR(255) UNIQUE NOT NULL,
    password_hash VARCHAR(255) NOT NULL,
    nickname VARCHAR(100),
    avatar_url VARCHAR(500),
    phone VARCHAR(20),
    country VARCHAR(10) DEFAULT 'TH',  -- 泰国/越南/印尼/马来西亚
    language VARCHAR(10) DEFAULT 'th',
    
    -- 认证相关
    email_verified BOOLEAN DEFAULT FALSE,
    verify_token VARCHAR(255),
    reset_token VARCHAR(255),
    reset_token_expires TIMESTAMP,
    
    -- 状态
    status VARCHAR(20) DEFAULT 'active',  -- active/suspended/banned
    last_login_at TIMESTAMP,
    last_login_ip VARCHAR(45),
    
    -- 时间戳
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    
    -- 索引
    CONSTRAINT users_email_unique UNIQUE (email)
);

CREATE INDEX idx_users_status ON users(status);
CREATE INDEX idx_users_created_at ON users(created_at);
```

#### 会员等级表 (members)

```sql
CREATE TABLE members (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    code VARCHAR(50) UNIQUE NOT NULL,  -- free/basic/pro/premium

    -- 定价
    price_monthly DECIMAL(10,2) DEFAULT 0,
    price_yearly DECIMAL(10,2) DEFAULT 0,
    currency VARCHAR(10) DEFAULT 'USD',
    
    -- 权益限制
    ai_request_limit INT,               -- 每月AI请求次数，NULL=无限制
    product_limit INT,                   -- 商品数量上限
    order_limit INT,                     -- 订单数量上限
    storage_limit_mb INT DEFAULT 100,   -- 存储空间MB
    enable_analytics BOOLEAN DEFAULT FALSE,
    enable_priority_support BOOLEAN DEFAULT FALSE,
    enable_api_access BOOLEAN DEFAULT FALSE,
    
    -- 描述
    description TEXT,
    features JSONB,                     -- 特性列表
    
    sort_order INT DEFAULT 0,
    is_active BOOLEAN DEFAULT TRUE,
    
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

#### 订阅表 (subscriptions)

```sql
CREATE TABLE subscriptions (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
```

```typescript
import crypto from 'crypto';

const ALGORITHM = 'aes-256-gcm';

function encrypt(text: string, key: Buffer): string {
  const iv = crypto.randomBytes(16);
  const cipher = crypto.createCipheriv(ALGORITHM, key, iv);
  
  let encrypted = cipher.update(text, 'utf8', 'hex');
  encrypted += cipher.final('hex');
```

```typescript
  const tier = getRateLimitTier(ctx.path);
```

#### Redis分布式限流

```typescript
// 滑动窗口限流算法
async function slidingWindowRateLimit(
  redis: Redis,
  key: string,
  limit: number,
  windowSeconds: number
): Promise<{ allowed: boolean; remaining: number }> {
  const now = Date.now();
  const windowStart = now - windowSeconds * 1000;
  
  // 移除窗口外的请求
  await redis.zremrangebyscore(key, 0, windowStart);
  
  // 获取当前请求数
  const count = await redis.zcard(key);
  
  if (count >= limit) {
    return { allowed: false, remaining: 0 };
  }
  
  // 添加新请求
  await redis.zadd(key, now, `${now}-${Math.random()}`);
  await redis.expire(key, windowSeconds);
  
  return { allowed: true, remaining: limit - count - 1 };
}
```

```typescript
// API签名验证 (可选，用于高风险接口)
function verifyApiSignature(params: Record<string, string>, secret: string): boolean {
  const timestamp = params.timestamp;
  const signature = params.signature;
  
  // 验证时间戳 (5分钟内)
  if (Date.now() - parseInt(timestamp) > 5 * 60 * 1000) {
    return false;
  }
  
  // 验证签名
  const expectedSignature = crypto
    .createHmac('sha256', secret)
    .update(`${timestamp}${JSON.stringify(params)}`)
    .digest('hex');
  
  return signature === expectedSignature;
}
```

### 4.2 数据加密方案

#### 4.2.1 传输层加密

| 项目 | 配置 | 说明 |
|------|------|------|
| **TLS版本** | TLS 1.2 minimum, TLS 1.3 preferred | 禁用TLS 1.0/1.1 |

| **数据库** | 透明数据加密 (TDE) | 云服务商KMS |

```typescript
const maskingRules = {
  // 手机号: 138****5678
  phone: (value: string) => value.replace(/(\d{3})\d{4}(\d{4})/, '$1****$2'),
  
  // 邮箱: t***@example.com
  email: (value: string) => {
    const [name, domain] = value.split('@');
    return `${name[0]}***@${domain}`;
  },
  
  // 身份证: 110***1234
  idCard: (value: string) => value.replace(/(\d{3})\d{11}(\d{4})/, '$1***$2'),
  
  // 银行卡: **** **** **** 1234
  bankCard: (value: string) => `**** **** **** ${value.slice(-4)}`,
  
  // 姓名: 张*三
  name: (value: string) => value.replace(/(.).(.)/, '$1*$2'),
};
```

### 5.1 数据库设计原则（便于迁移）

#### 5.1.1 云厂商无关设计

```prisma
// 优先使用标准SQL，避免厂商特有功能
model User {
  id        String   @id @default(uuid())  // UUID优于自增ID
  email     String   @unique
  createdAt DateTime @default(now())       // 标准时间戳
  updatedAt DateTime @updatedAt
  
  @@index([email])
  @@map("users")  // 显式表名映射
}

// 避免使用：
// - MySQL: YEAR() 函数
// - PostgreSQL特有: JSONB路径操作
// - 特定云厂商: Supabase Storage路径格式
```

#### 5.1.2 迁移友好原则

| 原则 | 说明 | 实现方式 |
|------|------|----------|
| **避免自增ID** | 跨数据库冲突 | 使用UUID |
| **标准时间戳** | 时区一致性 | UTC + ISO 8601 |
| **显式表名** | 避免默认命名 | @@map() |
| **最小关联** | 减少迁移复杂度 | 适度反范式 |
| **配置外置** | 环境差异隔离 | 环境变量 |

### 5.2 数据导出/导入方案

#### 5.2.1 导出格式

| 格式 | 适用场景 | 工具 |
|------|----------|------|
| **JSON** | 全量数据、小规模 | Prisma + jq |
| **CSV** | 结构化数据、批量导入 | psql COPY |
| **SQL** | 完整迁移 | pg_dump/mysqldump |
| **Parquet** | 大数据分析 | Spark/DataBricks |

#### 5.2.2 导出流程

```bash
#!/bin/bash
# 数据导出脚本

# 1. 导出前准备
echo "Starting data export at $(date)"
BACKUP_DIR="/backups/$(date +%Y%m%d)"
mkdir -p $BACKUP_DIR

# 2. 导出用户数据
pg_dump -h $DB_HOST -U $DB_USER -d $DB_NAME \
  --table=users \
  --format=custom \
  --file=$BACKUP_DIR/users.dump

# 3. 导出业务数据
pg_dump -h $DB_HOST -U $DB_USER -d $DB_NAME \
  --table=products,orders \
  --format=custom \
  --file=$BACKUP_DIR/business.dump

# 4. 生成导出清单
cat > $BACKUP_DIR/manifest.json <<EOF
{
  "exportDate": "$(date -Iseconds)",
  "version": "1.0",
  "checksums": {
    "users": "$(sha256sum $BACKUP_DIR/users.dump)",
    "business": "$(sha256sum $BACKUP_DIR/business.dump)"
  }
}
EOF

# 5. 加密备份
gpg --symmetric --cipher-algo AES256 $BACKUP_DIR/*.dump
```
