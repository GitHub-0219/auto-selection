## 一、项目概述

### 1.1 项目背景

Auto选品是科技管理集团旗下的核心产品，专注于为TikTok Shop东南亚市场卖家提供智能化选品解决方案。品牌Slogan：「让出海，像呼吸一样简单」。

当前项目处于早期发展阶段，需要搭建清晰的公司架构以支持：
- 后续融资需求
- 知识产权保护
- 业务合规运营
- 税务优化规划

### 1.2 架构设计原则

| 原则 | 说明 |
|------|------|
| 风险隔离 | 各公司职能分离，单一风险不传导 |
| 资产保护 | 核心资产集中在特定主体 |
| 融资友好 | 便于投资人进入和退出 |
| 税务优化 | 充分利用海南自贸港政策 |
| 控制权保障 | 创始人保持实际控制 |

## 三、股权结构设计

### 3.1 各公司股权比例

| 公司 | 股东 | 持股比例 | 备注 |
|------|------|----------|------|
| **HoldCo** | 创始人团队 | 100% | 海南自贸港注册 |
| **InvCo** | HoldCo | 100% | 融资主体 |
| **TechCo** | HoldCo | 70% | 知识产权主体 |
| | InvCo | 30% | 投资人持股预留 |
| **OpCo** | HoldCo | 100% | 运营主体 |

### 3.2 股权分离方案

#### 3.2.1 股权与经营权分离

```
创始人 → HoldCo（100%）→ InvCo/TechCo/OpCo
                        ↓
                   创始人担任董事/高管
                   日常经营权下放专业团队
```

**分离要点：**
- HoldCo层面保留绝对控制权
- 运营层面引入职业经理人
- 投资人通过InvCo参与重大决策

#### 3.2.2 股权激励安排

| 激励池 | 来源 | 比例 | 归属对象 |
|--------|------|------|----------|
| 期权池 | InvCo | 15% | 核心员工/顾问 |
| 预留股 | TechCo | 10% | 未来核心人才 |

### 3.3 控制权保障机制

#### 3.3.1 股权层面控制

| 控制机制 | 具体措施 |
|----------|----------|
| 绝对控股 | 创始人通过HoldCo持有各公司100%或70%以上股权 |
| 一票否决权 | 章程约定重大事项需要HoldCo同意 |
| 董事指派权 | HoldCo保留各公司多数董事席位 |

#### 3.3.2 协议层面控制

| 协议类型 | 控制要点 |
|----------|----------|
| 一致行动人协议 | 投资人股权由HoldCo代为行使表决权 |
| 优先认购权 | 后续融资时保持持股比例 |
| 反稀释条款 | 保护创始团队不被过度稀释 |

#### 3.3.3 运营层面控制

```
创始人 → CEO/执行董事 → 各公司日常运营
              ↓
        技术负责人（TechCo）
        运营负责人（OpCo）
        融资负责人（InvCo）
```

## 五、融资路径设计

### 5.1 InvCo融资主体设计

#### 5.1.1 融资架构

```
投资人 → InvCo → 持有TechCo/OpCo股权
              ↓
         享受投资收益
         参与重大决策
```

#### 5.1.2 融资阶段规划

| 阶段 | 融资轮次 | 目标金额 | 估值参考 | 出让股权 |
|------|----------|----------|----------|----------|
| 种子轮 | 天使/种子 | 100-300万 | 1000-2000万 | 10-15% |
| Pre-A | Pre-A | 500-1000万 | 3000-5000万 | 10-20% |
| A轮 | A轮 | 2000-5000万 | 1-2亿 | 15-25% |
| B轮+ | B轮及后续 | 待定 | 待定 | 待定 |

### 5.2 投资人进入路径

#### 5.2.1 股权进入

```
Step 1: 创始人注册HoldCo（海南自贸港）
Step 2: HoldCo全资设立InvCo、TechCo、OpCo
Step 3: 投资人以增资扩股方式进入InvCo
Step 4: InvCo使用资金认购TechCo/OpCo新增股权
Step 5: 完成工商变更，投资人穿透持有项目权益
```

#### 5.2.2 可转债进入

```
适用场景：早期估值不确定
结构：投资人 → 出借资金 → 转为TechCo/OpCo股权
优势：灵活定价，降低早期估值风险
```

#### 5.2.3 SAF进入（可选）

```
适用场景：快速小额融资
文件：Simple Agreement for Future Equity
特点：简化流程，快速到账
```

### 5.3 退出机制

#### 5.3.1 退出路径

| 退出方式 | 说明 | 适用场景 |
|----------|------|----------|
| **IPO退出** | 上市后减持 | 最终退出目标 |
| **并购退出** | 被大厂收购 | 战略价值实现 |
| **股权转让** | 老股转让 | 投资人中途退出 |
| **回购退出** | 公司/创始人回购 | 保障投资人权益 |
| **清算退出** | 公司注销清算 | 项目失败 |

#### 5.3.2 退出优先级

```
第一顺位：并购退出（如果战略价值高）
第二顺位：IPO退出（长期目标）
第三顺位：股权转让（灵活安排）
第四顺位：回购退出（保底安排）
```

## 六、风险隔离设计

### 6.1 运营风险隔离

#### 6.1.1 OpCo运营风险隔离

```
HoldCo（资产保护层）
    │
    └── TechCo（知识产权层）
              │
              └── OpCo（运营风险层）
                        │
                        ├── 客户纠纷
                        ├── 平台合规风险
                        └── 运营亏损
```

**隔离措施：**
- OpCo以有限责任公司运营
- 亏损不影响HoldCo/TechCo资产
- 业务合同由OpCo签署，不穿透至母公司

#### 6.1.2 客户数据风险隔离

| 风险类型 | 隔离措施 |
|----------|----------|
| 数据泄露 | OpCo仅存储必要数据，敏感数据加密 |
| 合规风险 | 聘请当地法务，TikTok Shop合规运营 |
| 投诉纠纷 | OpCo独立承担，HoldCo不承担连带责任 |

### 6.2 融资风险隔离

#### 6.2.1 投资人风险隔离

```
投资人 → InvCo（隔离层）→ TechCo/OpCo
               │
               ├── 以认缴资本为限承担责任
               ├── 不影响个人创始人资产
               └── 投资协议明确风险承担
```

**隔离措施：**
- 投资人入股InvCo，不直接持有项目公司
- 投资协议明确「有限责任公司」责任边界
- 创始人个人资产与公司资产分离

#### 6.2.2 对赌风险隔离

| 条款类型 | 设计要点 |
|----------|----------|
| 业绩对赌 | 设定合理目标，避免无法达成 |
| 上市对赌 | 设置较长周期，降低短期压力 |
| 退出对赌 | 明确回购触发条件和价格上限 |

### 6.3 分红风险隔离

#### 6.3.1 分红路径

```
OpCo盈利 → 分红给TechCo（许可费抵扣）
              │
              ▼
         TechCo → 分红给InvCo
              │
              ▼
         InvCo → 分红给HoldCo
              │
              ▼
         HoldCo → 分红给创始人
```

#### 6.3.2 分红限制

| 限制类型 | 具体措施 |
|----------|----------|
| 法定限制 | 利润弥补亏损后提取10%公积金 |
| 运营限制 | 预留3-6个月运营资金 |
| 融资限制 | 投资协议约定分红比例上限 |

## 七、税务优化设计

### 7.1 海南自贸港政策利用

#### 7.1.1 企业所得税优惠

| 政策内容 | 适用条件 | 节省税负 |
|----------|----------|----------|
| 15%企业所得税率 | 海南自贸港注册企业 | 相比25%节省40% |
| 鼓励类产业目录 | 科技/贸易类企业 | 符合条件可申请 |
| 新增境外直接投资 |免税| 境外投资收益免税 |

#### 7.1.2 个人所得税优惠

| 政策内容 | 适用条件 | 节省税负 |
|----------|----------|----------|
| 最高15%个税税率 | 高端人才/紧缺人才 | 相比45%最高档大幅降低 |
| 人才认定 | 学历/职称/收入等条件 | 需符合海南标准 |

#### 7.1.3 增值税与其他优惠

| 税种 | 优惠政策 | 适用场景 |
|------|----------|----------|
| 增值税 | 部分商品零税率 | 出口服务 |
| 关税 | 免征进口关税 | 自用设备进口 |
| 印花税 | 减免优惠 | 海南区域内交易 |

### 7.2 跨境交易税务设计

#### 7.2.1 技术授权税务架构

```
TechCo（海南）→ 授权 → OpCo/合作方
                      │
                      ├── 许可费支付
                      ├── 跨境付汇
                      └── 代扣代缴预提税
```

**税务优化要点：**
- TechCo开具增值税发票给OpCo
- 跨境许可费代扣代缴预提税（10%）
- 利用海南自贸港政策降低整体税负

#### 7.2.2 关联交易定价

| 交易类型 | 定价原则 | 税务考量 |
|----------|----------|----------|
| 技术授权 | 市场公允价值 | 避免被认定为转移定价 |
| 服务提供 | 成本加成法 | 5-15%加成比例 |
| 资金往来 | 独立交易原则 | 避免利息税务风险 |

#### 7.2.3 东南亚市场税务考量

| 国家 | 预提税税率 | 税务协定 | 优化建议 |
|------|------------|----------|----------|
| 泰国 | 15% | 有 | 申请税收抵免 |
| 越南 | 10% | 有 | 利用协定优惠 |
| 马来西亚 | 10% | 有 | 设立中间控股 |
| 印尼 | 20% | 无 | 设立新加坡中间公司 |
