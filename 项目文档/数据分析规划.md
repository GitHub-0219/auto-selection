## 一、核心指标体系

### 1.1 指标框架（PIRATE模型）

```
┌─────────────────────────────────────────────────────────────┐
│                      PIRATE模型                              │
├─────────────────────────────────────────────────────────────┤
│  P - Promotion  拉新指标                                      │
│  I - Activation 激活指标                                      │
│  R - Retention  留存指标                                      │
│  A - Revenue    营收指标                                      │
│  T - Referral   推荐指标                                      │
│  E - Effect     效果指标                                      │
└─────────────────────────────────────────────────────────────┘
```

### 1.2 核心指标定义

| 指标名称 | 定义 | 计算方式 | 目标值 | 监控频率 |
|----------|------|----------|--------|----------|
| **DAU** | 日活跃用户 | 当日登录用户数 | 500+ | 每日 |
| **WAU** | 周活跃用户 | 7日内有活跃的用户数 | 2000+ | 每周 |
| **MAU** | 月活跃用户 | 30日内有活跃的用户数 | 8000+ | 每月 |
| **DAU/MAU** | 用户活跃度 | DAU/MAU×100% | ≥15% | 每周 |
| **新增用户数** | 每日新增注册 | 当日新注册用户数 | 100+ | 每日 |
| **次留率** | 次日留存率 | Day1留存/Day0新增×100% | ≥40% | 每日 |
| **7日留存率** | 7日留存率 | Day7留存/Day0新增×100% | ≥20% | 每周 |
| **30日留存率** | 30日留存率 | Day30留存/Day0新增×100% | ≥10% | 每月 |
| **付费转化率** | 付费用户占比 | 付费用户/总用户×100% | ≥5% | 每月 |
| **ARPU** | 用户平均收入 | 总收入/活跃用户 | ¥30+ | 每月 |
| **LTV** | 用户生命周期价值 | ARPU×平均生命周期 | ¥360+ | 每季度 |

## 二、关键业务指标

### 2.1 选品模块指标

| 指标 | 定义 | 计算方式 | 目标 |
|------|------|----------|------|
| **选品报告生成量** | 每日生成的选品报告数 | SUM(生成次数) | 200+ |
| **报告生成成功率** | 成功生成/总请求 | 成功数/总数×100% | ≥95% |
| **报告生成时长** | 从点击到结果返回时间 | AVG(完成时间-开始时间) | ≤60秒 |
| **选品报告分享率** | 分享报告的用户占比 | 分享用户/生成用户×100% | ≥10% |
| **报告后行为率** | 查看报告后执行下一步 | 收藏/导出/进入内容创作 | ≥30% |

### 2.2 内容创作模块指标

| 指标 | 定义 | 计算方式 | 目标 |
|------|------|----------|------|
| **内容生成量** | 每日生成的内容数 | SUM(生成次数) | 500+ |
| **生成成功率** | 成功生成/总请求 | 成功数/总数×100% | ≥98% |
| **内容导出率** | 导出内容的用户占比 | 导出用户/生成用户×100% | ≥50% |
| **生成满意度** | 用户对生成结果的评分 | AVG(评分) | ≥4.0/5.0 |

### 2.3 培训课程指标

| 指标 | 定义 | 计算方式 | 目标 |
|------|------|----------|------|
| **课程观看人数** | 观看课程的用户数 | COUNT(DISTINCT user_id) | 1000+ |
| **课程完播率** | 完成观看的比例 | 看完用户/开始用户×100% | ≥60% |
| **课程完成率** | 完成全部课程的用户比例 | 完成用户/开始用户×100% | ≥30% |
| **课程转化率** | 观看后付费的用户比例 | 付费用户/观看用户×100% | ≥8% |
| **学习时长** | 用户平均学习时长 | AVG(总学习时长) | ≥30分钟 |

### 2.4 账号诊断指标

| 指标 | 定义 | 计算方式 | 目标 |
|------|------|----------|------|
| **诊断使用量** | 每日诊断次数 | SUM(诊断次数) | 50+ |
| **诊断付费率** | 付费诊断占比 | 付费诊断/总诊断×100% | ≥20% |
| **诊断后优化率** | 执行建议的用户比例 | 执行用户/诊断用户×100% | ≥40% |

## 三、转化漏斗

### 3.1 用户转化漏斗

```
┌─────────────────────────────────────────────────────────────────┐
│                        用户全链路转化漏斗                            │
└─────────────────────────────────────────────────────────────────┘

                           访问落地页
                              │
                         100% │ 浏览用户
                              ▼
                        ┌──────────┐
                        │ 注册账号  │ ← 目标：≥8%
                        └────┬─────┘
                             │ 8%
                        ┌────┴─────┐
                        │ 完成引导  │ ← 目标：≥70%
                        └────┬─────┘
                             │ 5.6%
                        ┌────┴─────┐
                        │ 首次选品  │ ← 目标：≥50%
                        └────┬─────┘
                             │ 2.8%
                        ┌────┴─────┐
                        │ 首次创作  │ ← 目标：≥40%
                        └────┬─────┘
                             │ 1.1%
                        ┌────┴─────┐
                        │ 观看课程  │ ← 目标：≥60%
                        └────┬─────┘
                             │ 0.7%
                        ┌────┴─────┐
                        │ 首次付费  │ ← 目标：≥5%
                        └────┬─────┘
                             │ 0.03%
                        ┌────┴─────┐
                        │ 续费用户  │ ← 目标：≥30%
                        └──────────┘
```

### 3.2 各阶段关键指标

| 阶段 | 核心指标 | 计算方式 | 健康值 | 预警值 |
|------|----------|----------|--------|--------|
| **注册** | 注册转化率 | 注册用户/浏览用户×100% | ≥8% | <5% |
| **引导** | 引导完成率 | 完成引导/注册用户×100% | ≥70% | <50% |
| **激活** | 首次选品率 | 选品用户/注册用户×100% | ≥50% | <30% |
| **使用** | 深度使用率 | 生成3次+内容/活跃用户×100% | ≥30% | <20% |
| **付费** | 付费转化率 | 付费用户/注册用户×100% | ≥5% | <3% |
| **留存** | 30日留存率 | Day30留存/Day0新增×100% | ≥10% | <5% |

## 四、数据埋点方案

### 4.1 埋点规范

```
命名规范：[模块]_[动作]_[属性]

示例：
- user_register_success（用户注册成功）
- product_select_click（商品选择点击）
- report_generate_complete（报告生成完成）
```

### 4.2 核心埋点清单

#### 4.2.1 用户模块

| 事件名称 | 触发时机 | 携带参数 |
|----------|----------|----------|
| `user_app_open` | 打开APP/进入官网 | platform, channel, version |
| `user_register_click` | 点击注册按钮 | register_type(手机/微信) |
| `user_register_success` | 注册成功 | register_type, user_id |
| `user_login_success` | 登录成功 | login_type, user_id |
| `user_profile_complete` | 完成资料设置 | profile_type, completion_rate |

#### 4.2.2 选品模块

| 事件名称 | 触发时机 | 携带参数 |
|----------|----------|----------|
| `select_page_view` | 进入选品页面 | source |
| `select_condition_set` | 设置筛选条件 | market, budget, category |
| `select_analysis_click` | 点击开始分析 | market_list, condition_count |
| `select_report_generate` | 报告生成完成 | generate_time, result_count |
| `select_report_view` | 查看报告详情 | report_id, view_duration |
| `select_product_click` | 点击推荐商品 | product_id, product_name |
| `select_product_collect` | 收藏商品 | product_id |
| `select_export_click` | 导出报告 | export_type, report_id |

#### 4.2.3 内容创作模块

| 事件名称 | 触发时机 | 携带参数 |
|----------|----------|----------|
| `create_page_view` | 进入创作页面 | source |
| `create_type_select` | 选择创作类型 | create_type(Listing/脚本) |
| `create_input_complete` | 完成信息输入 | input_type, field_count |
| `create_generate_click` | 点击生成按钮 | create_type, market |
| `create_generate_success` | 生成成功 | generate_time, result_count |
| `create_copy_click` | 复制内容 | content_type |
| `create_export_click` | 导出内容 | export_type |

#### 4.2.4 课程模块

| 事件名称 | 触发时机 | 携带参数 |
|----------|----------|----------|
| `course_page_view` | 进入课程页面 | source |
| `course_chapter_click` | 点击课程章节 | course_id, chapter_id |
| `course_play_start` | 开始播放视频 | course_id, chapter_id |
| `course_play_complete` | 播放完成 | course_id, chapter_id, watch_duration |
| `course_test_start` | 开始课后测试 | course_id |
| `course_test_complete` | 完成课后测试 | course_id, score, pass_status |

#### 4.2.5 支付模块

| 事件名称 | 触发时机 | 携带参数 |
|----------|----------|----------|
| `pay_page_view` | 进入支付页面 | product_id, product_type |
| `pay_click` | 点击购买按钮 | product_id, price |
| `pay_success` | 支付成功 | order_id, product_id, amount |
| `pay_fail` | 支付失败 | product_id, fail_reason |
| `subscribe_renew` | 续费成功 | product_id, renew_count |

### 5.2 核心看板设计

#### 5.2.1 实时监控看板

| 模块 | 指标 | 刷新频率 | 展示形式 |
|------|------|----------|----------|
| **流量概览** | DAU、实时在线、新增用户 | 5分钟 | 数字+趋势图 |
| **实时转化** | 当日转化漏斗进度 | 5分钟 | 漏斗图 |
| **告警监控** | 异常事件提醒 | 实时 | 列表+弹窗 |

#### 5.2.2 运营看板

| 模块 | 指标 | 更新频率 | 展示形式 |
|------|------|----------|----------|
| **用户概览** | DAU/MAU/WAU | 每日 | 折线图+对比 |
| **留存看板** | 次留/7留/30留曲线 | 每日 | 留存矩阵 |
| **转化看板** | 各环节转化率 | 每日 | 漏斗图+趋势 |
| **内容看板** | 生成量/成功率/满意度 | 每日 | 趋势图+排名 |

#### 5.2.3 财务看板

| 模块 | 指标 | 更新频率 | 展示形式 |
|------|------|----------|----------|
| **收入概览** | 今日收入/本月收入/环比 | 每日 | 数字+趋势 |
| **付费分析** | 付费用户数/付费率/ARPU | 每日 | 趋势图 |
| **订单分析** | 订单数/客单价/订单分布 | 每日 | 图表 |
| **预测看板** | 月度收入预测 | 每周 | 预测图 |

### 5.3 数据看板访问权限

| 角色 | 可访问看板 |
|------|------------|
| CEO/创始人 | 全部看板 |
| 产品负责人 | 全部看板 |

| 数据分析师 | 全部看板 |

```
┌─────────────────────────────────────────────────────────────────┐
│                        数据采集架构                               │
└─────────────────────────────────────────────────────────────────┘

                        ┌──────────────┐
                        │   用户端      │

                    │                     │
```

## 七、数据驱动机制

### 7.1 日常监控机制

| 时间 | 内容 | 负责人 |
|------|------|--------|
| 9:00 | 发送日报数据摘要 | 数据分析师 |
| 10:00 | 查看告警列表 | 值班人员 |
| 18:00 | 汇总当日核心指标 | 运营负责人 |
| 周一 | 发送周报数据 | 数据分析师 |

### 7.2 数据复盘机制

| 周期 | 内容 | 参与人 |
|------|------|--------|
| 每周 | 产品数据分析复盘 | 产品/运营/技术 |
| 每月 | 月度业务分析会议 | 全员 |
| 每季度 | 战略规划数据对齐 | 管理层 |

### 7.3 数据预警机制

| 预警级别 | 触发条件 | 通知方式 | 响应时间 |
|----------|----------|----------|----------|
| P0-紧急 | 核心功能不可用 | 电话+短信 | 5分钟 |
| P1-严重 | DAU下降>30% | 钉钉/企微 | 30分钟 |
| P2-警告 | 转化率下降>20% | 邮件 | 2小时 |
| P3-提示 | 指标偏离阈值 | 看板标记 | 次日处理 |

## 八、数据安全规范

### 8.1 数据分级

| 级别 | 定义 | 示例 | 保护措施 |
|------|------|------|----------|
| **L1-公开** | 可对外公开 | 产品功能介绍 | 无限制 |
| **L2-内部** | 公司内部使用 | 运营数据 | 权限控制 |
| **L3-敏感** | 用户隐私数据 | 手机号/地址 | 加密+审计 |
| **L4-机密** | 核心商业数据 | 收入数据/成本 | 最高权限 |

### 8.2 权限管理

```
权限层级：
- 超级管理员：全部权限
- 数据管理员：数据管理+报表配置
- 分析师：数据查询+导出
- 普通用户：查看基础看板
- 访客：仅查看公开数据
```
