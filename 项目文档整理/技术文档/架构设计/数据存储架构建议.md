# 数据存储架构建议

> 科技管理集团 - 数据安全与迁移方案

## 一、数据分类

### 1.1 按重要程度

| 级别 | 类型 | 数据示例 | 存储策略 |
|------|------|----------|----------|
| **核心数据** | 业务数据 | 用户信息、订单、支付记录 | 多地备份+加密 |
| **重要数据** | 内容数据 | 课程内容、Prompt模板 | 定期备份 |
| **一般数据** | 日志数据 | 访问日志、操作日志 | 定期归档 |
| **临时数据** | 缓存数据 | Redis缓存、Session | 无需备份 |

### 1.2 按敏感程度

| 级别 | 类型 | 数据示例 | 加密要求 |
|------|------|----------|----------|
| **机密** | 身份数据 | 密码、支付信息、身份证 | 强加密(AES-256) |
| **敏感** | 业务数据 | 用户行为、交易记录 | 加密存储 |
| **内部** | 运营数据 | 课程、Prompt、配置 | 权限控制 |
| **公开** | 公开内容 | 产品介绍、帮助文档 | 无加密 |

## 二、存储架构推荐

### 2.1 推荐方案：云原生架构

```
┌─────────────────────────────────────────────────────────┐
│                    应用层                                │
│              NestJS / Vue3 应用                         │
└─────────────────────────────────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────────┐
│                    数据层                                │
│  ┌──────────┐  ┌──────────┐  ┌──────────┐             │
│  │PostgreSQL│  │  Redis   │  │ MongoDB  │             │
│  │ (主数据) │  │ (缓存)   │  │ (日志)   │             │
│  └──────────┘  └──────────┘  └──────────┘             │
└─────────────────────────────────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────────┐
│                    备份层                                │
│  ┌──────────┐  ┌──────────┐  ┌──────────┐             │
│  │ 自动备份  │  │ 异地备份  │  │ 对象存储  │             │
│  │ (每日)   │  │ (每周)   │  │ (OSS)    │             │
│  └──────────┘  └──────────┘  └──────────┘             │
└─────────────────────────────────────────────────────────┘
```

### 2.2 数据库选型

| 数据库 | 用途 | 为什么选择 |
|--------|------|------------|
| **PostgreSQL** | 核心业务数据 | ACID保证、JSON支持、成熟稳定 |
| **Redis** | 缓存/Session | 高性能、支持持久化 |
| **MongoDB** | 日志/非结构化 | 灵活Schema、写入高性能 |

## 三、备份策略

### 3.1 自动备份方案

| 类型 | 频率 | 保留时间 | 存储位置 |
|------|------|----------|----------|
| **全量备份** | 每日凌晨2点 | 30天 | 对象存储(OSS) |
| **增量备份** | 每小时 | 7天 | 本地+OSS |
| **实时备份** | 实时 | - | 主从同步 |

### 3.2 备份脚本示例

```bash
#!/bin/bash
# 每日自动备份脚本

DATE=$(date +%Y%m%d)
BACKUP_DIR="/backup/$DATE"
DB_NAME="auto_selection"

# 创建备份目录
mkdir -p $BACKUP_DIR

# PostgreSQL备份
pg_dump -U postgres $DB_NAME | gzip > $BACKUP_DIR/db_$DATE.sql.gz

# 上传到OSS
aliyun oss cp $BACKUP_DIR oss://your-bucket/backup/$DATE/ -r

# 清理30天前的备份
find /backup -type d -mtime +30 -exec rm -rf {} \;

echo "Backup completed: $DATE"
```

### 3.3 备份验证

- **每周**：随机抽取备份文件进行恢复测试
- **每月**：完整恢复演练
- **记录**：备份验证报告

## 四、安全措施

### 4.1 访问控制

| 层面 | 措施 |
|------|------|
| **网络** | VPC隔离、安全组白名单 |
| **应用** | JWT认证、RBAC权限 |
| **数据库** | 独立账号、最小权限 |
| **加密** | TLS传输、AES存储加密 |

### 4.2 敏感数据处理

```
密码：bcrypt加密存储，不可逆
支付信息：不存储完整卡号，只存Token
身份证：AES-256加密，单独存储
日志：脱敏处理，不记录敏感信息
```

### 4.3 安全审计

- 所有数据库操作记录审计日志
- 敏感数据访问告警
- 定期安全扫描

## 五、迁移方案

### 5.1 迁移友好设计

| 设计 | 说明 |
|------|------|
| **容器化** | Docker部署，一键迁移 |
| **配置分离** | 环境变量管理，无硬编码 |
| **数据导出** | 标准SQL/JSON格式 |
| **版本控制** | 数据库Schema版本管理 |

### 5.2 迁移步骤

```
1. 导出数据
   - pg_dump导出PostgreSQL
   - redis-cli导出Redis
   - mongodump导出MongoDB

2. 迁移存储
   - 上传到对象存储
   - 或直接数据传输

3. 恢复数据
   - pg_restore恢复PostgreSQL
   - redis-cli恢复Redis
   - mongorestore恢复MongoDB

4. 验证数据
   - 数据完整性检查
   - 功能测试验证
```

### 5.3 迁移脚本

```bash
#!/bin/bash
# 数据迁移脚本

# 导出
pg_dump -U postgres auto_selection > export/db.sql
redis-cli --rdb export/redis.rdb
mongodump --db logs --out export/mongo

# 打包
tar -czf backup_$(date +%Y%m%d).tar.gz export/

# 迁移到新服务器
scp backup_*.tar.gz user@new-server:/backup/

# 恢复
pg_restore -U postgres -d auto_selection export/db.sql
redis-cli --pipe < export/redis.rdb
mongorestore export/mongo
```
