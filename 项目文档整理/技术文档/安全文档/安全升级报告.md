## 二、JWT密钥安全加固

### 2.1 问题分析

**原配置（不安全）**：
```bash
JWT_SECRET="demo-secret-key-for-testing-2024"
```

**风险评估**：
- ❌ 使用公开演示值，可被轻易猜测
- ❌ 密钥长度不足32字符
- ❌ 缺乏足够的熵值（随机性）
- ❌ 可能被记录在公开代码仓库

### 2.2 新密钥生成规范

**生成标准**：
```bash
# 技术要求
- 最小长度: 64字节 (512位)
- 字符集: 十六进制 (0-9, a-f)
- 随机源: 系统安全随机数生成器 (secrets module)
- 存储: 环境变量或密钥管理服务 (KMS)
```

**生成方法（Python示例）**：
```python
import secrets

# 方法1: 十六进制格式（推荐）
jwt_secret = secrets.token_hex(64)  # 128字符，512位熵

# 方法2: Base64格式（更长）
jwt_secret = secrets.token_urlsafe(64)  # 86字符，可选

# 方法3: 使用OpenSSL
openssl rand -hex 64
```

### 2.3 安全配置指南

**Step 1: 生成新密钥**
```bash
# 在生产服务器上执行，禁止在开发环境生成后硬编码
openssl rand -hex 64 > /tmp/jwt_secret.txt
chmod 600 /tmp/jwt_secret.txt
```

**Step 2: 配置环境变量**
```bash
# 方式A: 直接设置（临时）
export JWT_SECRET=$(cat /tmp/jwt_secret.txt)

# 方式B: 添加到 .env.production
echo 'JWT_SECRET=<生成的密钥>' >> .env.production

# 方式C: 使用Docker Secret
echo "${JWT_SECRET}" | docker secret create jwt_secret -
```

**Step 3: 更新配置文件**
```typescript
// src/app.module.ts
JwtModule.registerAsync({
  useFactory: (configService: ConfigService) => {
    const jwtSecret = configService.get<string>('JWT_SECRET')
    
    // 验证密钥强度
    if (!jwtSecret || jwtSecret.length < 64) {
      throw new Error('JWT_SECRET must be at least 64 characters')
    }
    
    // 检测是否为演示密钥
    const demoPatterns = ['demo', 'test', 'secret', 'password', '123456']
    if (demoPatterns.some(p => jwtSecret.toLowerCase().includes(p))) {
      throw new Error('JWT_SECRET cannot contain common weak patterns')
    }
    
    return {
      secret: jwtSecret,
      signOptions: { 
        expiresIn: '7d',
        algorithm: 'HS256'  // 生产环境建议迁移到 RS256
      },
    }
  },
  inject: [ConfigService],
})
```

**Step 4: 密钥轮换机制**
```typescript
// src/common/security/key-rotation.service.ts
@Injectable()
export class KeyRotationService {
  private readonly keyVersion: number = 1;
  private readonly rotationInterval = 90 * 24 * 60 * 60 * 1000; // 90天
  
  // 支持多版本密钥兼容
  async verifyToken(token: string): Promise<boolean> {
    const versions = [
      process.env.JWT_SECRET_V1,
      process.env.JWT_SECRET_V2,
      process.env.JWT_SECRET, // 当前密钥
    ].filter(Boolean);
    
    for (const secret of versions) {
      try {
        jwt.verify(token, secret);
        return true;
      } catch {
        continue;
      }
    }
    return false;
  }
}
```

## 三、数据库SSL配置指南

### 3.1 问题分析

**原配置（不安全）**：
```bash
DATABASE_URL="postgresql://postgres:postgres@localhost:5432/ai_cross_border?schema=public"
```

**风险评估**：
- ❌ 明文传输数据库凭证
- ❌ 传输层无加密，可被中间人攻击
- ❌ 不符合数据安全法规（如GDPR）
- ❌ 云环境部署存在数据泄露风险

### 3.2 SSL配置方案

#### 方案A: PostgreSQL 自签名证书（测试/内网）

**Step 1: 生成证书**
```bash
# 生成CA证书
openssl req -new -x509 -days 365 -nodes \
  -out ca.crt -keyout ca.key \
  -subj "/CN=PostgreSQL CA/O=AutoSelection"

# 生成服务器证书
openssl req -new -nodes \
  -out server.csr -keyout server.key \
  -subj "/CN=localhost/O=AutoSelection"

# 签署服务器证书
openssl x509 -req -days 365 \
  -in server.csr -CA ca.crt -CAkey ca.key \
  -CAcreateserial -out server.crt

# 设置权限
chmod 600 server.key ca.key
```

**Step 2: 配置 PostgreSQL**
```bash
# postgresql.conf
ssl = on
ssl_cert_file = '/path/to/server.crt'
ssl_key_file = '/path/to/server.key'
ssl_ca_file = '/path/to/ca.crt'

# pg_hba.conf - 强制SSL连接
hostssl all all 0.0.0.0/0 md5 clientcert=verify-full
```

**Step 3: 更新 Prisma 配置**
```bash
# .env.production
DATABASE_URL="postgresql://postgres:password@host:5432/ai_cross_border?schema=public&sslmode=require"
```

#### 方案B: AWS RDS / 云数据库（推荐生产环境）

**配置步骤**：
```bash
# 1. 下载 AWS RDS 根证书
wget https://truststore.pki.rds.amazonaws.com/global/global-bundle.pem

# 2. 配置环境变量
DATABASE_URL="postgresql://user:password@xxx.rds.amazonaws.com:5432/dbname?sslmode=require&sslrootcert=global-bundle.pem"

# 3. Prisma schema 配置
```

**Prisma Schema 更新**：

#### 方案C: 使用 PgBouncer 连接池 + SSL

## 四、其他安全漏洞修复

### 4.1 响应头安全加固

**修复前问题**：
- 可能泄露 `X-Powered-By` 等敏感头
- CSP策略在开发环境缺失

**修复后配置**：
```typescript
// src/common/security/security-headers.middleware.ts
@Injectable()
export class SecurityHeadersMiddleware implements NestMiddleware {
  use(req: Request, res: Response, next: NextFunction) {
    const env = process.env.NODE_ENV || 'development'
    
    // 1. 移除敏感信息头
    res.removeHeader('X-Powered-By')
    res.removeHeader('Server')
    res.removeHeader('X-AspNet-Version')
    
    // 2. 基础安全头
    res.setHeader('X-Content-Type-Options', 'nosniff')
    res.setHeader('X-Frame-Options', 'DENY')
    res.setHeader('X-XSS-Protection', '1; mode=block')
    res.setHeader('X-Download-Options', 'noopen')
    res.setHeader('X-Permitted-Cross-Domain-Policies', 'none')
    res.setHeader('Referrer-Policy', env === 'production' 
      ? 'strict-origin-when-cross-origin' 
      : 'no-referrer')
    
    // 3. 权限策略 (Permissions Policy)
    res.setHeader('Permissions-Policy', 
      'accelerometer=(), camera=(), geolocation=(), gyroscope=(), magnetometer=(), microphone=(), payment=()')
    
    // 4. 内容安全策略 (生产环境)
    if (env === 'production') {
      res.setHeader('Content-Security-Policy', [
        "default-src 'self'",
        "script-src 'self' 'unsafe-inline' 'unsafe-eval'",
        "style-src 'self' 'unsafe-inline'",
        "img-src 'self' data: https: blob:",
        "font-src 'self' data:",
        "connect-src 'self' https://api.autoselection.com wss://api.autoselection.com",
        "frame-ancestors 'none'",
        "form-action 'self'",
        "base-uri 'self'",
        "object-src 'none'"
      ].join('; '))
    }
    
    // 5. HSTS (仅HTTPS环境)
    if (env === 'production' && req.protocol === 'https') {
      res.setHeader('Strict-Transport-Security', 
        'max-age=31536000; includeSubDomains; preload')
    }
    
    // 6. 禁用缓存
    res.setHeader('Cache-Control', 'no-store, no-cache, must-revalidate, proxy-revalidate')
    res.setHeader('Pragma', 'no-cache')
    res.setHeader('Expires', '0')
    
    next()
  }
}
```

### 4.3 输入验证增强

```typescript
// src/common/validators/enhanced-validators.ts
import { registerDecorator, ValidationOptions, ValidatorConstraint, ValidatorConstraintInterface } from 'class-validator'

// JWT密钥强度验证
@ValidatorConstraint({ name: 'jwtSecretStrength', async: false })
export class JwtSecretStrengthConstraint implements ValidatorConstraintInterface {
  validate(value: any): boolean {
    if (typeof value !== 'string' || value.length < 64) {
      return false
    }
    
    // 排除常见弱模式
    const weakPatterns = [
      /^(demo|test|secret|password|123456|admin)/i,
      /^(abc|def|ghi|qwerty)/i,
      /^(january|february|march|april)/i,
      /(admin|root|user|pass)/i
    ]
    
    return !weakPatterns.some(p => p.test(value))
  }
  
  defaultMessage(): string {
    return 'JWT_SECRET must be at least 64 characters and not contain weak patterns'
  }
}

// 数据库URL安全验证
@ValidatorConstraint({ name: 'databaseUrlSecurity', async: false })
export class DatabaseUrlSecurityConstraint implements ValidatorConstraintInterface {
  validate(value: any): boolean {
    if (typeof value !== 'string') return false
    
    // 必须包含SSL参数
    if (!value.includes('sslmode')) {
      return false
    }
    
    // 生产环境禁止localhost
    if (process.env.NODE_ENV === 'production' && value.includes('localhost')) {
      return false
    }
    
    return true
  }
  
  defaultMessage(): string {
    return 'DATABASE_URL must include sslmode parameter and use production host in production'
  }
}
```

### 4.4 审计日志增强

```typescript
// src/common/security/audit-log.service.ts
@Injectable()
export class AuditLogService {
  private readonly sensitiveFields = [
    'password', 'token', 'secret', 'key', 
    'authorization', 'credential', 'ssn', 'creditCard'
  ]
  
  async logSecurityEvent(event: SecurityEvent): Promise<void> {
    const logEntry = {
      timestamp: new Date().toISOString(),
      eventType: event.type,
      severity: event.severity,
      ip: this.maskIp(event.ip),
      userId: event.userId || 'anonymous',
      path: event.path,
      method: event.method,
      userAgent: event.userAgent,
      // 敏感数据脱敏
      metadata: this.sanitize(event.metadata),
      // 附加上下文
      environment: process.env.NODE_ENV,
      region: process.env.AWS_REGION || 'unknown',
    }
    
    // 结构化日志输出
    if (event.severity === 'critical') {
      console.error(JSON.stringify(logEntry))
      // 触发告警
      await this.notifySecurityTeam(logEntry)
    } else {
      console.log(JSON.stringify(logEntry))
    }
  }
  
  private sanitize(data: any): any {
    if (!data) return data
    
    const sanitized = { ...data }
    for (const field of this.sensitiveFields) {
      if (sanitized[field]) {
        sanitized[field] = '***REDACTED***'
      }
    }
    return sanitized
  }
  
  private maskIp(ip: string): string {
    if (!ip) return 'unknown'
    const parts = ip.split('.')
    if (parts.length === 4) {
      return `${parts[0]}.${parts[1]}.xxx.xxx`
    }
    return 'masked'
  }
}
```
