## 🔒 GitHub Secrets 配置

推送前请确保配置以下 GitHub Secrets（如果启用 CI/CD）：

### 必需 Secrets
- [ ] `DOCKER_USERNAME` - Docker Hub 用户名
- [ ] `DOCKER_TOKEN` - Docker Hub Access Token

### 部署 Secrets（可选）
- [ ] `STAGING_HOST` - Staging 服务器地址
- [ ] `STAGING_USER` - Staging 服务器用户名
- [ ] `STAGING_SSH_KEY` - Staging 服务器 SSH 私钥
- [ ] `PRODUCTION_HOST` - Production 服务器地址
- [ ] `PRODUCTION_USER` - Production 服务器用户名
- [ ] `PRODUCTION_SSH_KEY` - Production 服务器 SSH 私钥
- [ ] `FRONTEND_API_URL` - 前端 API 地址

## 📋 推送前最终检查

```bash
# 1. 确认 .gitignore 正确
cat .gitignore

# 2. 确认敏感文件被忽略
git status --ignored

# 3. 检查是否有意外提交的 .env
git ls-files | grep "\.env$"

# 4. 确认所有配置文件已创建
ls -la .github/workflows/
ls -la .github/labeler.yml
ls -la LICENSE
ls -la CONTRIBUTING.md
ls -la README.md
```

## 🚀 安全最佳实践

1. **永远不要**提交真实的 API 密钥
2. 使用 `.env.example` 作为配置模板
3. 所有密钥存储在 GitHub Secrets 中
4. 生产环境**必须**更换所有默认密钥
5. 定期轮换 API 密钥
6. 使用最小权限原则配置第三方服务密钥
