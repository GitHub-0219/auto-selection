# Auto选品项目 - 反扒机制设计文档

## 一、设计背景

### 1.1 项目背景
Auto选品是一个TikTok跨境电商选品工具，核心数据包括：
- 市场趋势数据
- 竞品分析数据
- 选品推荐算法
- 用户行为数据

这些数据具有较高的商业价值，容易成为爬虫的目标。

### 1.2 威胁模型
| 威胁类型 | 严重程度 | 描述 |
|----------|----------|------|
| 恶意爬虫 | 高 | 批量抓取产品数据 |
| API滥用 | 高 | 超过正常用量的API调用 |
| 自动化工具 | 中 | 使用脚本/工具自动化操作 |
| DDoS攻击 | 高 | 大量请求导致服务不可用 |
| 数据窃取 | 高 | 通过漏洞获取敏感数据 |

## 三、详细设计

### 3.1 请求频率限制

#### 3.1.1 IP级别限流
```
配置: windowMs=60000, max=100

算法: 滑动窗口计数器
存储: Redis INCR + EXPIRE

响应:
{
  "success": false,
  "code": "RATE_LIMIT_IP",
  "message": "请求过于频繁，请稍后再试",
  "retryAfter": 60
}
```

#### 3.1.2 用户级别限流
```
配置: windowMs=60000, max=50

标识: userId > sessionId > IP
存储: Redis Hash
```

#### 3.1.3 API级别限流
```
配置: windowMs=60000, max=20

适用接口:
- /api/product/analyze
- /api/product/select
- /api/market/trend
- /api/competition/analysis

存储: Redis INCR per IP+Path
```

### 3.2 User-Agent检测

#### 3.2.1 黑名单UA列表
```javascript
const blockedUAs = [
  'scrapy',           // Python爬虫框架
  'python-requests',  // Python HTTP库
  'python-urllib',    // Python URL库
  'curl',             // 命令行工具
  'wget',             // 命令行下载
  'httpx',            // Python HTTP客户端
  'aiohttp',          // Python异步HTTP
  'node-fetch',       // Node.js fetch
  'axios/0.',         // Axios旧版本
  'postman',          // API测试工具
  'insomnia',         // API客户端
  'abot',             // 爬虫框架
  'semrush',          // SEO工具
  'ahrefs',           // SEO工具
  'majestic'          // SEO工具
];
```

#### 3.2.2 检测规则
```
1. 检查UA是否为空
2. 检查UA是否包含黑名单关键词（不区分大小写）
3. 符合任一条件返回403
```

### 3.3 验证码机制

#### 3.3.1 触发条件
| 条件 | 阈值 | 验证类型 |
|------|------|----------|
| IP请求频率 | >80次/分钟 | 图形验证码 |
| 用户请求频率 | >40次/分钟 | 图形验证码 |
| 异常行为检测 | 嫌疑度>50 | 滑块验证 |
| 新用户注册 | - | 滑块验证 |
| 敏感操作 | - | 短信验证 |

#### 3.3.2 验证码有效期
```
过期时间: 5分钟
最大验证尝试: 5次
验证通过有效期: 30分钟
```

### 3.4 行为分析

#### 3.4.1 检测指标
| 指标 | 机器人特征 | 权重 |
|------|-----------|------|
| 访问间隔方差 | <10ms | 30 |
| 鼠标移动速度方差 | <0.01 | 30 |
| 鼠标直线移动 | 方向变化<0.1rad | 25 |
| 点击间隔方差 | <100ms | 35 |
| 打字速度 | <50ms/字符 | 40 |
| Tab键使用比例 | >50% | 20 |
| 蜜罐字段触发 | 是 | 50 |

#### 3.4.2 嫌疑度阈值
```
0-30:   正常
31-50:  可疑 (触发验证码)
51-80:  高度可疑 (强制验证)
81-100: 机器人 (封禁IP)
>100:   封禁 + 上报
```

### 3.5 数据加密

#### 3.5.1 加密策略
```javascript
// 算法: AES-256-GCM
// 密钥: PBKDF2派生
// 格式: { encrypted, iv, authTag }
```

#### 3.5.2 脱敏字段
```javascript
const sensitiveFields = [
  'password', 'token', 'secret', 'apiKey',
  'phone', 'mobile', 'email',
  'idCard', 'bankCard', 'address', 'realName'
];
```

#### 3.5.3 脱敏规则
```
手机号: 138****5678
邮箱:   t***@example.com
银行卡: ****1234
身份证: 110***********1234
```

### 3.6 IP黑名单

#### 3.6.1 黑名单类型
| 类型 | 来源 | 有效期 |
|------|------|--------|
| 手动封禁 | 管理员操作 | 永久或自定义 |
| 自动封禁 | 异常行为触发 | 1-24小时 |
| 临时封禁 | 多次验证失败 | 5-60分钟 |
| 威胁情报 | 第三方情报源 | 动态更新 |

#### 3.6.2 支持的封禁格式
```
- 单个IP: 192.168.1.1
- IP范围: 192.168.1.0-192.168.1.255
- CIDR: 192.168.1.0/24
- IP段: 192.168.*.*
```

## 四、存储设计

### 4.1 Redis数据结构

```
# IP限流计数器
rate_limit:ip:{ip}          -> Counter (TTL: 60s)

# 用户限流计数器
rate_limit:user:{userId}    -> Counter (TTL: 60s)

# API限流计数器
rate_limit:api:{ip}:{path}  -> Counter (TTL: 60s)

# IP黑名单
blacklist:ip                -> Set
whitelist:ip               -> Set

# 验证码
captcha:{captchaId}         -> Hash { type, code, expiresAt, attempts }

# 行为数据
behavior:{sessionId}       -> Hash { clickCount, mouseMoves, suspicionLevel }

# 请求日志
access_log:{date}:{hour}   -> SortedSet { ip, score: timestamp }
```

### 4.2 内存缓存（备选）

```
ipStore     -> Map<IP, AccessRecord>
userStore   -> Map<UserID, AccessRecord>
blacklist   -> Set<IP>
captchaStore -> Map<CaptchaID, CaptchaData>
```

### 5.2 配置示例

```yaml
# config/anti-scraping.yaml
antiScraping:
  enabled: true
  
  rateLimit:
    ip:
      windowMs: 60000
      max: 100
    user:
      windowMs: 60000
      max: 50
    api:
      windowMs: 60000
      max: 20
      paths:
        - /api/product/analyze
        - /api/product/select
  
  ua:
    blockedPatterns:
      - scrapy
      - python-requests
      - curl
      - wget
    allowEmpty: false
  
  captcha:
    enabled: true
    expireTime: 300
    maxAttempts: 5
    triggerThreshold: 80
  
  blacklist:
    autoBlock:
      enabled: true
      threshold: 50
      duration: 3600000
  
  encryption:
    enabled: true
    algorithm: aes-256-gcm
```
