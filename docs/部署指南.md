## 三、后端配置

### 3.1 环境变量

创建 `.env.production` 文件：

```bash
# 反扒机制配置
RATE_LIMIT_IP_WINDOW_MS=60000
RATE_LIMIT_IP_MAX=100
RATE_LIMIT_USER_MAX=50
RATE_LIMIT_API_MAX=20

# 加密配置
SIGNATURE_SECRET=your-secure-signature-key-here
CAPTCHA_SECRET=your-captcha-secret-here
ENCRYPTION_KEY=your-encryption-key-here

# Redis配置
REDIS_HOST=127.0.0.1
REDIS_PORT=6379
REDIS_PASSWORD=

# 日志配置
LOG_LEVEL=info
LOG_FILE=./logs/anti-scraping.log
```

### 3.2 集成反扒中间件

在 `backend/app.js` 中添加：

```javascript
const express = require('express');
const { ipLimiter, userLimiter, apiLimiter, checkUserAgent, behaviorAnalysis } = require('./middleware/antiScraping');
const captchaRoutes = require('./routes/captcha');
const adminRoutes = require('./routes/admin');

const app = express();

// 全局中间件
app.use(checkUserAgent);  // UA检测
app.use(behaviorAnalysis); // 行为分析

// 路由级中间件
app.use('/api', ipLimiter());  // IP限流

// 验证码路由
app.use('/api/captcha', captchaRoutes);

// 管理接口
app.use('/api/admin', adminRoutes);

// 敏感API限流
const sensitivePaths = [
  '/api/product/analyze',
  '/api/product/select',
  '/api/market/trend'
];

app.use(sensitivePaths, apiLimiter());

// 用户级别限流（需要认证）
app.use('/api/user', userLimiter());
```

## 四、Redis配置

### 4.1 Redis安装

```bash
# Ubuntu/Debian
sudo apt update
sudo apt install redis-server

# 启动Redis
sudo systemctl start redis
sudo systemctl enable redis

# 测试连接
redis-cli ping
# 应返回: PONG
```

### 4.2 Redis配置优化

编辑 `/etc/redis/redis.conf`:

```conf
# 内存限制
maxmemory 256mb
maxmemory-policy allkeys-lru

# 持久化
save 900 1
save 300 10
save 60 10000

# 日志
loglevel notice
logfile /var/log/redis/redis-server.log
```

## 五、前端集成

### 5.1 引入行为检测

在 `frontend/lib/behaviorCheck.js` 引入:

```javascript
import { BehaviorDetector } from '../lib/behaviorCheck';

// 初始化行为检测
const detector = new BehaviorDetector({
  suspiciousClickThreshold: 20,
  suspiciousMoveThreshold: 500,
  idleTimeout: 30000
});

// 全局暴露
window.behaviorDetector = detector;

// 触发验证码
window.triggerCaptcha = function(options) {
  // 显示验证码UI
  showCaptchaModal(options.reason);
};
```

## 六、启动服务

### 6.1 PM2配置

创建 `ecosystem.config.js`:

```javascript
module.exports = {
  apps: [
    {
      name: 'auto-select-backend',
      script: 'backend/app.js',
      instances: 2,
      exec_mode: 'cluster',
      env: {
        NODE_ENV: 'production',
        PORT: 3000
      },
      error_file: './logs/backend-error.log',
      out_file: './logs/backend-out.log',
      time: true
    },
    {
      name: 'auto-select-frontend',
      script: 'npx',
      args: 'next start -p 8080',
      cwd: './frontend',
      instances: 1,
      env: {
        NODE_ENV: 'production'
      }
    }
  ]
};
```

### 6.2 启动命令

```bash
# 启动所有服务
pm2 start ecosystem.config.js

# 查看状态
pm2 status

# 查看日志
pm2 logs

# 重启服务
pm2 restart all

# 停止服务
pm2 stop all
```

## 十、故障排除

### 10.1 常见问题

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| 正常用户被拦截 | 限流阈值过低 | 调整 max 配置 |
| Redis连接失败 | Redis未启动 | systemctl start redis |
| 验证码无法显示 | 前端资源加载失败 | 检查静态资源路径 |
| 性能下降严重 | 未使用Redis | 部署Redis集群 |

### 10.2 日志分析

```bash
# 查看限流日志
grep "RATE_LIMIT" logs/anti-scraping.log

# 查看封禁日志
grep "IP_BLOCKED" logs/anti-scraping.log

# 查看验证码日志
grep "CAPTCHA" logs/anti-scraping.log
```
