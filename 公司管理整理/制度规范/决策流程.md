# 决策流程

## 一、决策理念

### 1.1 核心原则
- **DRI原则（最终责任人）**：每个决策必须有明确的最终责任人
- **速度优先**：在不确定性中快速决策
- **可逆决策快速**：大多数决策可以快速做出，错了再改
- **不可逆决策谨慎**：重大不可逆决策需充分论证
- **数据驱动**：用数据支撑决策，而非直觉

### 1.2 决策质量标准
| 标准 | 说明 |
|-----|-----|
| 正确性 | 决策是否解决了真正的问题 |
| 完整性 | 是否考虑了所有关键因素 |
| 可执行性 | 决策是否可以被执行 |
| 时效性 | 决策是否及时 |
| 清晰度 | 决策是否清晰可传达 |

## 二、决策分类

### 2.1 按重要性分类
| 类型 | 定义 | 决策层级 | 示例 |
|-----|-----|---------|-----|
| 战略决策 | 影响公司发展方向 | 董事会/CEO | 业务方向、融资、重大投资 |
| 战术决策 | 影响部门或业务线 | VP/总监 | 部门目标、资源分配 |
| 执行决策 | 日常业务执行 | 经理/主管 | 具体任务分配、日常审批 |
| 例行决策 | 标准流程内 | 任何人 | 常规报销、简单审批 |

### 2.2 按可逆性分类
| 类型 | 定义 | 决策速度 | 决策深度 |
|-----|-----|---------|---------|
| 高可逆 | 错了容易改 | 快 | 浅 |
| 中可逆 | 改造成本中等 | 中等 | 中等 |
| 低可逆 | 很难改变 | 慢 | 深 |

### 2.3 按影响范围分类
| 类型 | 影响范围 | 决策人 |
|-----|---------|-------|
| 公司级 | 全公司 | CEO/董事会 |
| 部门级 | 单个部门 | 部门负责人 |
| 项目级 | 单个项目 | 项目负责人 |
| 个人级 | 个人工作 | 个人 |

## 四、决策流程

### 4.1 标准决策流程（IDD模型）
```
Identify（识别） → Define（定义） → Decide（决策） → Do（执行） → Evaluate（评估）
```

#### 第一阶段：识别（Identify）
- 发现问题或机会
- 收集相关信息
- 初步判断是否需要决策

#### 第二阶段：定义（Define）
- 明确决策目标
- 确定决策边界
- 定义成功标准

#### 第三阶段：决策（Decide）
- 收集方案选项
- 分析评估选项
- 选择最优方案
- 确定行动计划

#### 第四阶段：执行（Do）
- 分解行动项
- 分配责任人
- 设定时间节点
- 执行并监控

#### 第五阶段：评估（Evaluate）
- 跟踪执行结果
- 评估决策效果
- 总结经验教训
- 必要时调整

### 4.2 决策流程规范
| 阶段 | 时间要求 | 产出物 |
|-----|---------|-------|
| 识别 | 即时 | 问题/机会描述 |
| 定义 | 1天内 | 决策简报 |
| 决策 | 按紧急程度 | 决策记录 |
| 执行 | 按计划 | 执行结果 |
| 评估 | 执行后1周 | 评估报告 |

## 五、决策方法

### 5.1 日常决策
**原则**：快速决策，75%正确率即可，不要过度分析

**方法**：
- 遵循优先级
- 参考已有政策
- 必要时请教上级
- 错了及时调整

### 5.2 重要决策
**原则**：充分论证，民主讨论，CEO最终决定

**方法**：
- 充分收集信息
- 多种方案比较
- 团队充分讨论
- DRI最终决定

### 5.3 重大决策
**原则**：严谨论证，风险评估，集体决策

**方法**：
1. 成立专项小组
2. 深度调研分析
3. 多轮讨论评审
4. 董事会决策

### 5.4 决策工具推荐
| 工具 | 适用场景 |
|-----|---------|
| 六顶思考帽 | 多角度分析问题 |
| 决策树 | 风险评估 |
| 利弊分析 | 方案比较 |
| 波特五力 | 战略分析 |
| SWOT分析 | 战略分析 |
| AARRR模型 | 增长决策 |

## 六、会议决策

### 6.1 决策会议类型
| 类型 | 适用场景 | 时长 | 参与人 |
|-----|---------|-----|-------|
| 快速决策会 | 紧急事项 | 30分钟 | 相关人 |
| 标准决策会 | 一般事项 | 60分钟 | DRI+顾问 |
| 重大决策会 | 战略事项 | 90分钟+ | 管理层 |

### 6.2 决策会议规范
| 环节 | 时长 | 说明 |
|-----|-----|-----|
| 陈述 | 20% | 各方陈述观点 |
| 讨论 | 50% | 充分讨论 |
| 总结 | 20% | DRI总结 |
| 决定 | 10% | 明确决策 |

### 6.3 共识达成
| 方式 | 说明 | 适用场景 |
|-----|-----|---------|
| DRI决定 | DRI一个人做决定 | 时间紧迫 |
| 多数同意 | 超过半数同意 | 一般事项 |
| 共识决策 | 所有人同意 | 需全员执行 |
| 保留意见 | 多数通过，允许保留 | 分歧较大 |

## 九、决策效率提升

### 9.1 提升决策效率的方法
| 方法 | 说明 |
|-----|-----|
| 设定决策时限 | 规定决策必须在多少时间内完成 |
| 授权下放 | 充分授权，减少决策上报 |
| 预定义规则 | 对常见事项预定义决策规则 |
| 减少决策数量 | 合并同类决策，减少决策频率 |
| 信息前置 | 提前准备信息，减少决策等待 |

### 9.2 决策效率指标
| 指标 | 定义 | 目标 |
|-----|-----|-----|
| 决策周期 | 从提出到决策的平均时间 | < 3天 |
| 决策执行率 | 决策被执行的比例 | > 90% |
| 决策成功率 | 决策达成预期目标的比例 | > 70% |
| 决策升级率 | 需要上级决策的比例 | < 20% |

## 十、决策文化

### 10.1 好的决策文化
- **敢于决策**：不推诿、不逃避
- **善于决策**：用方法、用数据
- **快速决策**：不拖沓、不犹豫
- **执行到位**：说了就做，做了就成
- **善于复盘**：错了改，改了进步

### 10.2 决策禁忌
- **不决策**：遇到问题绕着走
- **慢决策**：犹豫不决，错失良机
- **独断决策**：不听意见，一意孤行
- **情绪决策**：被情绪左右判断
- **完美决策**：追求完美，延误时机

### 10.3 DRI原则详解
**Definition（定义）**：
- 每个决策只有一个DRI
- DRI对决策负全部责任
- 其他人都只是顾问或执行者

**Application（应用）**：
- DRI可以是个人或角色
- 重大决策DRI通常是CEO
- 日常决策DRI是一线负责人

**Benefit（好处）**：
- 责任清晰，不推诿
- 决策高效，不等待
- 便于复盘，有追溯
