# 沟通协作规范

## 一、沟通理念

### 1.1 核心原则（参考阿里"视人为人"）
- **简单直接**：有问题直接说，不拐弯抹角
- **开放透明**：信息共享，减少信息差
- **尊重高效**：尊重他人时间，高效沟通
- **结果导向**：沟通服务于目标达成

### 1.2 沟通价值观
| 价值观 | 含义 | 行为表现 |
|-------|-----|---------|
| 客户第一 | 对内外部客户都要用心 | 及时响应，主动服务 |
| 开放协作 | 打破壁垒，共享信息 | 主动分享，积极配合 |
| 直接透明 | 有话直说，信息对称 | 坦诚表达，不藏着 |
| 尊重专业 | 尊重不同领域的专业性 | 不越权，尊重判断 |

## 二、信息同步机制

### 2.1 信息同步类型
| 类型 | 同步方式 | 同步频率 | 接收范围 |
|-----|---------|---------|---------|
| 公司动态 | 全员邮件/公告 | 按需 | 全员 |
| 项目进展 | 周报/看板 | 每周 | 项目相关方 |
| 业务数据 | 数据报表 | 每日/每周 | 管理层 |
| 紧急事项 | 即时通讯 | 发生时 | 相关方 |
| 变更通知 | 邮件/会议 | 发生时 | 相关方 |

### 2.2 信息同步渠道
| 渠道 | 用途 | 响应时效 |
|-----|-----|---------|
| 邮件 | 正式沟通、需要记录 | 24小时内 |
| 即时通讯（飞书） | 日常沟通、快速协作 | 2小时内 |
| 文档/知识库 | 信息沉淀、知识共享 | 按需查阅 |
| 会议 | 讨论决策、需要同步 | 按会议约定 |
| 周报/月报 | 工作汇报、状态同步 | 按周期 |

### 2.3 信息分级
| 级别 | 定义 | 传递方式 | 接收人 |
|-----|-----|---------|-------|
| P0紧急 | 影响业务的紧急问题 | 电话/即时通讯 | 相关方+升级 |
| P1重要 | 需要尽快处理 | 即时通讯 | 相关方 |
| P2常规 | 正常工作沟通 | 邮件/即时通讯 | 相关方 |
| P3信息 | 仅供参考 | 邮件/文档 | 订阅人 |

## 三、跨部门协作

### 3.1 协作原则
| 原则 | 说明 |
|-----|-----|
| 共同目标 | 以公司目标为导向，而非部门利益 |
| 主动沟通 | 提前沟通，不等问题升级 |
| 责任共担 | 共同对结果负责 |
| 互为客户 | 把其他部门当内部客户 |

### 3.2 协作机制
| 机制 | 说明 | 频率 | 参与人 |
|-----|-----|-----|-------|
| 部门周会 | 各部门内部同步 | 每周 | 部门全员 |
| 产品对齐会 | 产品技术运营对齐 | 每周 | 产品+技术+运营 |
| 跨部门同步会 | 跨部门问题协调 | 每两周 | 各部门负责人 |
| 项目周会 | 项目进展同步 | 每周 | 项目团队 |
| 问题协调会 | 紧急问题处理 | 按需 | 相关方 |

### 3.3 协作流程
```
需求提出 → 需求确认 → 方案评审 → 计划制定 → 执行协作 → 成果交付 → 反馈评价
```

| 阶段 | 规范 | 产出物 |
|-----|-----|-------|
| 需求提出 | 书面提出，清晰描述 | 需求文档 |
| 需求确认 | 接收方确认理解 | 确认记录 |
| 方案评审 | 共同讨论，确定方案 | 方案文档 |
| 计划制定 | 明确时间、人力 | 项目计划 |
| 执行协作 | 定期同步，解决问题 | 进展报告 |
| 成果交付 | 按时交付，验收确认 | 交付物 |
| 反馈评价 | 相互评价，持续改进 | 反馈记录 |

### 3.4 协作问题处理
| 问题类型 | 处理方式 | 升级路径 |
|---------|---------|---------|
| 理解偏差 | 重新沟通确认 | 直接沟通 |
| 资源冲突 | 部门负责人协调 | 项目负责人 |
| 优先级冲突 | 管理层裁决 | CEO |
| 战略分歧 | 战略对齐会 | 董事会 |

## 四、问题上报机制

### 4.1 上报原则
| 原则 | 说明 |
|-----|-----|
| 及时上报 | 问题发生后尽快上报 |
| 准确描述 | 清晰描述问题现象 |
| 初步分析 | 提供初步原因分析 |
| 建议方案 | 提出解决建议 |

### 4.2 上报流程
```
问题发现 → 初步处理 → 影响评估 → 上报 → 协调解决 → 反馈
```

### 4.3 上报标准
| 级别 | 定义 | 上报时间 | 上报人 | 接收人 |
|-----|-----|---------|-------|-------|
| L1 | 个人可解决 | 不上报 | - | - |
| L2 | 团队内解决 | 4小时内 | 个人 | 团队负责人 |
| L3 | 部门协调 | 2小时内 | 团队负责人 | 部门负责人 |
| L4 | 跨部门协调 | 1小时内 | 部门负责人 | CEO |
| L5 | 公司级影响 | 立即 | 发现人 | CEO |

### 4.4 上报模板
```markdown
## 问题上报

### 问题描述
[清晰描述问题现象]

### 问题级别
[L1-L5]

### 影响范围
[影响什么业务/多少人]

### 初步分析
[可能的原因]

### 已采取措施
[已经做了什么]

### 需要什么支持
[资源/决策/协助]
```

## 五、日常沟通规范

### 5.1 即时通讯规范（飞书）
| 场景 | 规范 |
|-----|-----|
| 工作时间 | 8:00-22:00，其他时间非紧急不用 |
| 消息格式 | 清晰简洁，分段表达 |
| @人 | 只@需要回复的人 |
| 已读回执 | 重要消息开启 |
| 群消息 | 尽量减少大群，聚焦讨论 |
| 文件传输 | 大文件用云盘链接 |

### 5.2 邮件沟通规范
| 规范 | 说明 |
|-----|-----|
| 主题清晰 | 明确邮件目的 |
| 内容简洁 | 结论前置，分点描述 |
| 附件说明 | 说明附件内容 |
| 抄送恰当 | 只抄送需要知悉的人 |
| 回复时效 | 24小时内回复 |
| 避免邮件群 | 讨论在即时通讯 |

**邮件模板**：
```
主题：[类型] 简要描述

邮件正文：

一、背景/目的
二、具体内容
三、行动项/请求
四、截止时间

---
附件：
```

### 5.3 面对面沟通
| 场景 | 规范 |
|-----|-----|
| 预约时间 | 提前预约，不要突然打扰 |
| 准备充分 | 提前想好要说什么 |
| 简明扼要 | 说重点，不废话 |
| 做好记录 | 记录关键结论和行动 |

### 5.4 电话沟通
| 场景 | 规范 |
|-----|-----|
| 确认时间 | 提前预约，不要突然打电话 |
| 自报家门 | 接通后先自我介绍 |
| 确认方便 | 询问对方是否方便 |
| 长话短说 | 控制通话时长 |
| 总结确认 | 通话结束前总结要点 |

## 六、会议沟通

### 6.1 会议前沟通
| 准备项 | 要求 |
|-------|-----|
| 了解议程 | 提前了解会议讨论什么 |
| 准备发言 | 准备好要说什么 |
| 准备材料 | 需要带什么材料 |
| 准时参会 | 按会议时间准时参加 |

### 6.2 会议中沟通
| 规范 | 说明 |
|-----|-----|
| 发言有序 | 举手或按顺序发言 |
| 表达清晰 | 观点明确，逻辑清晰 |
| 认真倾听 | 不打断别人发言 |
| 做好记录 | 记录重要信息 |
| 控制时间 | 不超时 |

### 6.3 会议后跟进
| 跟进项 | 要求 |
|-------|-----|
| 会议纪要 | 24小时内发出会议纪要 |
| 行动项执行 | 按会议决定执行行动项 |
| 进展反馈 | 按约定时间反馈进展 |

### 7.2 信息流规范
```
工作信息 → 即时通讯（飞书）
正式通知 → 邮件
协作文档 → 飞书文档
待办任务 → 项目管理系统
沉淀知识 → 知识库
```

### 7.3 消息礼仪
| 场景 | 建议 |
|-----|-----|
| 发消息后对方未回复 | 等待或追问，不要连环轰炸 |
| 对方已读不回 | 私信确认是否收到 |
| 紧急事项 | 说明"紧急"，或打电话 |
| 长消息 | 分段发送，重点加粗 |
| 需要对方决策 | 明确问句，提供选项 |

## 八、冲突处理

### 8.1 冲突类型
| 类型 | 说明 | 处理方式 |
|-----|-----|---------|
| 观点分歧 | 对问题看法不同 | 充分讨论，求同存异 |
| 资源竞争 | 资源有限，需要分配 | 优先级裁决 |
| 职责不清 | 边界模糊，互相推诿 | 明确职责，重新定义 |
| 利益冲突 | 部门或个人利益冲突 | 管理层协调 |

### 8.2 冲突处理原则
| 原则 | 说明 |
|-----|-----|
| 对事不对人 | 关注问题本身，不人身攻击 |
| 换位思考 | 理解对方立场 |
| 寻求共识 | 找共同目标 |
| 引入第三方 | 无法解决时寻求帮助 |

### 8.3 冲突处理流程
```
察觉冲突 → 私下沟通 → 充分讨论 → 寻求共识 → 达成一致
     ↓
无法解决 → 升级协调 → 管理层裁决
```

### 8.4 冲突升级
| 级别 | 定义 | 升级对象 |
|-----|-----|---------|
| 初级 | 同事间分歧 | 直接沟通 |
| 中级 | 团队间冲突 | 双方负责人 |
| 高级 | 部门间冲突 | CEO协调 |

## 九、远程协作

### 9.1 远程协作原则
| 原则 | 说明 |
|-----|-----|
| 异步优先 | 减少实时会议 |
| 文档化 | 重要信息文档沉淀 |
| 透明共享 | 信息充分透明 |
| 明确期望 | 明确交付时间和标准 |
| 定期同步 | 保持团队连接 |

### 9.2 远程协作规范
| 规范 | 说明 |
|-----|-----|
| 及时响应 | 工作时间2小时内响应 |
| 在线状态 | 工作时间保持在线 |
| 会议准备 | 提前测试设备 |
| 工作时间 | 保持核心时间重叠 |

### 9.3 远程会议规范
| 规范 | 说明 |
|-----|-----|
| 开启摄像头 | 建议开启，展示专注 |
| 发言示意 | 发言前举手上麦 |
| 静音管理 | 不发言时静音 |
| 会议纪要 | 安排专人记录 |

## 十、沟通能力提升

### 10.1 沟通技巧
| 技巧 | 说明 |
|-----|-----|
| 结论先行 | 先说结论，再说原因 |
| 分点表达 | 用1、2、3分点说明 |
| 具体明确 | 避免模糊表达 |
| 换位思考 | 考虑对方理解和感受 |
| 主动确认 | 确认对方理解正确 |

### 10.2 沟通能力评估
| 维度 | 评估项 |
|-----|-------|
| 表达能力 | 清晰准确、分点表达 |
| 倾听能力 | 理解对方、确认反馈 |
| 应变能力 | 灵活应对、恰当回应 |
| 协调能力 | 化解冲突、达成共识 |
| 影响能力 | 说服他人、推动执行 |

### 10.3 沟通培训
| 类型 | 形式 | 频率 |
|-----|-----|-----|
| 入职培训 | 新人沟通规范培训 | 入职时 |
| 技能培训 | 沟通技巧培训 | 按需 |
| 案例分享 | 优秀沟通案例分享 | 每季度 |
