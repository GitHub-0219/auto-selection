# 文档管理规范

## 一、文档理念

### 1.1 核心理念
- **知识沉淀**：文档是公司最重要的知识资产
- **信息透明**：文档是信息共享的载体
- **高效协作**：文档是团队协作的基础
- **持续积累**：文档需要持续更新维护

### 1.2 文档价值
| 价值 | 说明 |
|-----|-----|
| 知识传承 | 避免人员流动导致知识丢失 |
| 效率提升 | 减少重复沟通，提高效率 |
| 质量保障 | 规范化的文档确保质量 |
| 协作基础 | 跨地域、跨时区协作的保障 |

---

## 二、文档分类

### 2.1 按用途分类
| 类别 | 说明 | 示例 |
|-----|-----|-----|
| 战略文档 | 公司战略规划类 | 三年战略、年度规划 |
| 制度文档 | 管理制度规范类 | 财务制度、HR制度 |
| 产品文档 | 产品设计开发类 | PRD、技术文档 |
| 运营文档 | 业务运营操作类 | SOP、运营手册 |
| 项目文档 | 项目过程管理类 | 项目计划、复盘报告 |
| 知识文档 | 知识沉淀分享类 | 经验总结、最佳实践 |

### 2.2 按性质分类
| 类别 | 说明 | 管理方式 |
|-----|-----|---------|
| 正式文档 | 正式发布，需要版本管理 | 严格管理 |
| 工作文档 | 日常工作使用 | 常规管理 |
| 参考文档 | 仅供参考，随时可能更新 | 轻量管理 |
| 临时文档 | 短期使用 | 用后清理 |

### 2.3 按密级分类
| 级别 | 说明 | 访问权限 | 示例 |
|-----|-----|---------|-----|
| 公开 | 所有人都可以查看 | 全员 | 公司介绍 |
| 内部 | 仅公司内部可见 | 全员 | 制度文档 |
| 机密 | 核心机密信息 | 管理层+相关人 | 财务数据、融资信息 |
| 绝密 | 最高机密 | CEO+指定人 | 核心技术、商业秘密 |

### 3.2 命名规范
| 规则 | 说明 | 正确示例 | 错误示例 |
|-----|-----|---------|---------|
| 结构化命名 | 按一定结构命名 | `2026年产品路线图.md` | `产品路线图最新最终版.md` |
| 日期格式 | 包含日期用 YYYYMMDD | `20260101_会议纪要.md` | `1月1日会议记录.md` |
| 版本标注 | 正式文档标注版本 | `V1.0_XXX.md` | `XXX_v2_final_真的最后版.md` |
| 无空格 | 不使用空格分隔 | `Auto选品_用户手册.md` | `Auto选品 用户手册.md` |
| 中文可用 | 支持中文命名 | `产品需求文档.md` | - |
| 避免特殊字符 | 不使用特殊字符 | `项目立项模板.md` | `项目立项模板(新).md` |

### 3.3 命名模板
| 文档类型 | 命名模板 | 示例 |
|---------|---------|-----|
| 会议纪要 | `会议纪要_YYYYMMDD_主题.md` | `会议纪要_20260101_产品规划.md` |
| 周报 | `周报_YYYYWNN_姓名.md` | `周报_2026W01_张三.md` |
| 项目文档 | `项目_项目名_文档类型.md` | `项目_Auto选品_需求文档.md` |
| 制度文档 | `制度_制度名_V版本.md` | `制度_财务制度_V1.0.md` |
| 复盘报告 | `复盘_时间/事件_类型.md` | `复盘_2026Q1_产品发布.md` |

## 四、文档编写规范

### 4.1 文档结构
```
# 文档标题

## 一、概述
[简要说明文档目的和背景]

## 二、主体内容
[按逻辑组织的内容]

## 三、附录
[补充信息]

---
[文档信息]
- 版本：
- 更新日期：
- 作者：
```

### 4.2 文档内容规范
| 规范 | 说明 |
|-----|-----|
| 标题明确 | 标题清晰表达内容 |
| 格式统一 | 使用统一格式 |
| 内容完整 | 包含必要的上下文 |
| 更新及时 | 内容与实际同步 |
| 重点突出 | 使用加粗、列表等 |
| 图表辅助 | 适当使用图表说明 |

### 4.3 文档语言规范
| 规范 | 示例 |
|-----|-----|
| 使用主动语态 | "我们做了..." 而非 "被做了..." |
| 简洁明了 | 删除不必要的词 |
| 专业术语 | 使用统一的术语 |
| 避免歧义 | 表达清晰，不产生误解 |
| 一致性 | 同类内容表达一致 |

### 4.4 Markdown格式规范
| 元素 | 规范 |
|-----|-----|
| 标题层级 | 使用 # ## ### 层级不超过4层 |
| 列表 | 统一使用 - 或 1. |
| 链接 | 使用 [文字](URL) 格式 |
| 图片 | 使用相对路径或云存储 |
| 代码 | 使用 \`代码\` 或代码块 |
| 表格 | 使用标准Markdown表格 |

## 五、版本管理

### 5.1 版本命名
| 版本格式 | 说明 | 示例 |
|---------|-----|-----|
| V主版本.次版本 | 正式版本 | V1.0, V2.0 |
| V主版本.次版本.修订号 | 详细版本 | V1.0.1 |
| V主版本.次版本_状态 | 草稿版本 | V1.0_Draft |
| 日期版本 | 快速迭代文档 | 20260101 |

### 5.2 版本状态
| 状态 | 说明 | 标记 |
|-----|-----|-----|
| 草稿(Draft) | 正在编写 | `V1.0_Draft` |
| 评审(Review) | 等待评审 | `V1.0_Review` |
| 正式(Published) | 已发布正式版 | `V1.0` |
| 废弃(Deprecated) | 已废弃 | `V1.0_ Deprecated` |

### 5.3 版本记录
| 变更类型 | 记录要求 |
|---------|---------|
| 新增 | 记录新增内容 |
| 修改 | 记录修改内容 |
| 删除 | 记录删除内容 |
| 重大变更 | 详细说明变更原因 |

### 5.4 版本变更记录模板
```markdown
## 版本历史

| 版本 | 日期 | 作者 | 变更说明 |
|-----|-----|-----|---------|
| V1.0 | 2026-01-01 | 张三 | 初始版本 |
| V1.1 | 2026-02-01 | 李四 | 新增XXX，修改YYY |
| V2.0 | 2026-03-01 | 张三 | 重大版本升级，ZZZ |
```

## 六、文档权限

### 6.1 权限层级
| 级别 | 权限 | 说明 |
|-----|-----|-----|
| 查看 | 只读 | 可以查看内容 |
| 编辑 | 可修改 | 可以修改内容 |
| 管理 | 完全控制 | 可以删除、移动 |

### 6.2 权限分配原则
| 文档类型 | 创建人 | 默认权限 |
|---------|-------|---------|
| 个人文档 | 创建人 | 仅自己可编辑 |
| 团队文档 | 团队负责人 | 团队可编辑 |
| 公司文档 | 行政部门 | 全员可查看 |
| 机密文档 | CEO | 指定人可查看/编辑 |

### 6.3 权限审批
| 场景 | 审批人 |
|-----|-------|
| 普通文档权限申请 | 文档所有者 |
| 机密文档权限申请 | CEO |
| 删除重要文档 | 部门负责人 |

## 七、文档生命周期

### 7.1 文档生命周期
```
创建 → 评审 → 发布 → 使用 → 更新/废弃
```

| 阶段 | 说明 | 管理要求 |
|-----|-----|---------|
| 创建 | 编写文档初稿 | 遵循规范 |
| 评审 | 评审文档质量 | 需要审批 |
| 发布 | 正式发布使用 | 通知相关方 |
| 使用 | 日常使用维护 | 保持更新 |
| 更新 | 根据需要更新 | 版本管理 |
| 废弃 | 文档不再使用 | 归档处理 |

### 7.2 文档维护
| 类型 | 更新频率 | 负责人 |
|-----|---------|-------|
| 制度文档 | 年度回顾 | 制度制定部门 |
| 产品文档 | 按版本更新 | 产品/技术 |
| 项目文档 | 项目结束时归档 | 项目经理 |
| 知识文档 | 持续更新 | 知识贡献者 |
| 模板文档 | 按需更新 | 模板管理员 |

### 7.3 文档归档与清理
| 类型 | 保存期限 | 处理方式 |
|-----|---------|---------|
| 重要文档 | 永久 | 归档保存 |
| 项目文档 | 项目结束后+3年 | 归档保存 |
| 工作文档 | 2年 | 定期清理 |
| 临时文档 | 1年 | 到期清理 |

## 八、文档协作

### 8.1 协作规范
| 规范 | 说明 |
|-----|-----|
| 避免冲突 | 编辑前确认无他人编辑 |
| 及时保存 | 频繁保存，避免丢失 |
| 通知相关方 | 大幅修改需通知 |
| 说明变更 | 变更时说明做了什么 |

### 8.2 评审流程
| 场景 | 评审人 | 通过标准 |
|-----|-------|---------|
| 制度文档 | CEO | 内容准确、可执行 |
| 产品文档 | 产品+技术 | 完整、清晰 |
| 重要文档 | 相关方 | 无异议 |

### 8.3 评审清单
```
□ 文档目的明确
□ 内容完整准确
□ 格式规范统一
□ 逻辑清晰易读
□ 无错别字/错误
□ 相关方已确认
```
