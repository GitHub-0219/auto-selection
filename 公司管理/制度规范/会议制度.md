# 会议制度

## 一、会议理念（参考亚马逊"两个披萨原则"）

> **核心理念**：会议应该高效、务实、有结论
> - 精简参会人数，控制在"两个披萨能喂饱"的规模
> - 明确会议目的，确保每次会议都有产出
> - 尊重参与者时间，守时、聚焦

---

## 三、会议效率保障

### 3.1 会议准备规范
| 准备项 | 要求 | 责任方 |
|-------|-----|-------|
| 会议议程 | 提前24小时发出 | 主持人 |
| 会议资料 | 提前12小时发出 | 参会者 |
| 参会确认 | 提前12小时确认 | 所有参会人 |
| 会议通知 | 提前4小时发出 | 主持人 |

### 3.2 会议议程模板
```markdown
## 会议名称
**时间**：YYYY-MM-DD HH:MM - HH:MM
**地点**：线上/线下
**主持人**：
**参会人**：

### 议程
| 序号 | 议题 | 时长 | 主讲人 | 目标 |
|-----|-----|-----|-------|-----|
| 1 | | 分钟 | | |
| 2 | | 分钟 | | |

### 会前准备
- [ ] 资料1
- [ ] 资料2

### 会后跟进
| 行动项 | 负责人 | 截止时间 |
|-------|-------|---------|
| | | |
```

### 3.3 会议进行规范
| 规则 | 说明 |
|-----|-----|
| 准时开始 | 迟到超过5分钟，自动开始 |
| 准时结束 | 严格控制时间，不超时 |
| 聚焦主题 | 只讨论议程内事项 |
| 得出结论 | 每个议题必须有明确结论 |
| 记录要点 | 记录关键决策和行动项 |
| 指定主持人 | 每场会议必须明确主持人 |

### 3.4 会议时间控制
| 环节 | 建议时长 | 提醒 |
|-----|---------|-----|
| 开场/回顾 | 10% | - |
| 核心讨论 | 70% | 剩余10分钟提醒 |
| 总结/行动计划 | 20% | - |

## 五、会议类型规范

### 5.1 每日站会（Daily Standup）
| 项目 | 规范 |
|-----|-----|
| 时间 | 每天固定时间（如9:30） |
| 时长 | 15分钟，准时结束 |
| 形式 | 站立，快速简洁 |
| 内容 | 昨天完成、今天计划、阻碍问题 |
| 规则 | 不讨论细节，站立进行 |

**站会发言模板**：
```
大家好，我是XXX
昨天：完成了XXX
今天：计划做XXX
阻碍：XXX（需要帮助）
```

### 5.2 周例会（Weekly Meeting）
| 项目 | 规范 |
|-----|-----|
| 时间 | 每周一 9:30-10:30 |
| 时长 | 60分钟 |
| 参与者 | 全体员工 |
| 形式 | 线下/线上同步 |

**周例会流程**：
```
1. 公司/部门周报同步（10分钟）
2. 各部门上周总结+本周计划（30分钟）
3. 跨部门协调事项（10分钟）
4. 重要事项通报（5分钟）
5. 自由讨论（5分钟）
```

### 5.3 产品对齐会
| 项目 | 规范 |
|-----|-----|
| 时间 | 每周固定时间 |
| 时长 | 45分钟 |
| 参与者 | 产品、技术、运营负责人 |

**对齐会内容**：
- 产品需求评审
- 技术方案对齐
- 运营反馈同步
- 问题协调解决

### 5.4 Sprint评审/回顾会
| 项目 | Sprint评审 | Sprint回顾 |
|-----|-----------|------------|
| 时间 | 每两周周五下午 | 每两周周五下午 |
| 时长 | 60分钟 | 45分钟 |
| 参与者 | 研发+产品+相关方 | 研发团队 |
| 目的 | 演示成果，获取反馈 | 改进流程 |

### 5.5 决策评审会
| 项目 | 规范 |
|-----|-----|
| 触发条件 | 重大决策需求 |
| 准备要求 | 必须提前提交决策材料 |
| 时长 | 60-90分钟 |
| 决策原则 | 负责人决策制 |

**决策材料模板**：
```
## 决策事项
## 背景与问题
## 方案选项
   - 方案A：利弊分析
   - 方案B：利弊分析
   - 方案C：利弊分析
## 推荐方案及理由
## 所需资源
## 风险评估
## 建议决策
```

## 六、会议优化建议

### 6.1 提升会议效率的方法
| 方法 | 说明 |
|-----|-----|
| 取消不必要的会议 | 定期审视，取消无价值会议 |
| 缩小参会范围 | 只邀请必要人员 |
| 站立开会 | 促使会议简短 |
| 准时结束 | 培养守时文化 |
| 减少PPT | 鼓励讨论式会议 |
| 明确结论 | 每会必有结论和行动 |

### 6.2 判断会议必要性的问题
- 这个会议的目标是什么？
- 不开会能否达到目标？
- 是否有更高效的方式？
- 必须现场讨论吗？
- 能否用异步方式解决？

### 6.3 会议减少策略
| 替代方式 | 适用场景 |
|---------|---------|
| 异步文档 | 信息同步、状态更新 |
| 即时消息 | 简单问题、快速确认 |
| 邮件 | 正式沟通、需要记录 |
| 视频/语音 | 需讨论但不需面对面 |

## 七、会议礼仪

### 7.1 基本礼仪
| 礼仪 | 要求 |
|-----|-----|
| 守时 | 准时参加，准时结束 |
| 准备 | 提前了解议程，准备发言 |
| 专注 | 认真倾听，不做无关事情 |
| 尊重 | 尊重发言者，不打断 |
| 手机 | 静音或震动，紧急回复除外 |

### 7.2 线上会议礼仪
| 礼仪 | 要求 |
|-----|-----|
| 摄像头 | 建议开启，展示专注 |
| 网络 | 确保网络稳定 |
| 环境 | 选择安静环境 |
| 静音 | 不发言时静音 |
| 发言 | 发言前示意，避免抢话 |

```
[会议类型]_[会议日期]_[简短主题]

周例会_2026W01_第1周工作同步.md
```

## 九、会议效果评估

### 9.1 会议效果指标
| 指标 | 定义 | 目标 |
|-----|-----|-----|
| 会议数量 | 每周人均会议数 | < 3个 |
| 会议时长 | 每周人均会议时长 | < 3小时 |
| 会议结论率 | 有明确结论的会议比例 | > 90% |
| 行动项完成率 | 会议行动项按时完成率 | > 80% |
| 会议满意度 | 参会者满意度评分 | > 4.0/5 |

- 每月统计会议效率指标
- 季度回顾会议制度执行情况
- 根据反馈持续优化会议形式
