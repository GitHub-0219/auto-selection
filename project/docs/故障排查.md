## 部署问题

### 1. Node.js 版本不兼容

**错误信息**：
```
Error: The engine "node" is incompatible with this module
```

**原因**：Node.js 版本过低或过高

**解决方案**：
```bash
# 检查当前版本
node -v

# 使用 n 安装指定版本
npm install -g n
n 18

# 重新验证
node -v  # 应显示 v18.x.x
```

---

### 2. npm 安装依赖失败

**错误信息**：
```
npm ERR! code ETARGET
npm ERR! Unable to resolve dependency
```

**解决方案**：
```bash
# 清除 npm 缓存
npm cache clean --force

# 删除 node_modules 和 package-lock.json
rm -rf node_modules package-lock.json

# 重新安装
npm install
```

---

### 3. Git 克隆仓库中文目录乱码

**错误信息**：
```
fatal: cannot cd to '项目': No such file or directory
```

**原因**：Git 不支持中文目录名

**解决方案**：
```bash
# 在本地修复后重新推送
mv 项目 project
git add -A
git commit -m "refactor: rename Chinese folder to English"
git push

# 或在服务器上修复
mv 项目 project
sed -i 's|项目/|project/|g' .gitignore
```

---

### 4. 端口被占用

**错误信息**：
```
Error: listen EADDRINUSE :::3001
```

**解决方案**：
```bash
# 方法1：查找占用端口的进程
sudo lsof -i :3001

# 方法2：查看所有 node 进程
ps aux | grep node

# 终止占用进程
sudo kill -9 <PID>

# 或更换端口
export PORT=3002
```

---

### 5. PM2 启动失败

**错误信息**：
```
Error: Module not found: ./dist/main.js
```

**原因**：未执行构建步骤

**解决方案**：
```bash
cd backend
npm run build
pm2 start dist/main.js --name "backend"
```

---

### 6. Nginx 配置错误

**错误信息**：
```
nginx: [emerg] unexpected "}" in /etc/nginx/sites-enabled/...
```

**解决方案**：
```bash
# 检查语法
sudo nginx -t

# 查看详细错误
sudo cat /var/log/nginx/error.log

# 重新创建配置文件
sudo nano /etc/nginx/sites-available/auto-selection
```

正确配置示例：
```nginx
server {
    listen 80;
    server_name _;
    
    location / {
        root /var/www/auto-selection/frontend;
        index index.html;
        try_files $uri $uri/ /index.html;
    }
    
    location /api {
        proxy_pass http://127.0.0.1:3001;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
```

## 运行问题

### 1. 后端服务无法启动

**症状**：执行 `npm start` 后立即退出

**排查步骤**：
```bash
# 1. 检查环境变量
cat backend/.env

# 2. 检查端口占用
netstat -tlnp | grep 3001

# 3. 查看详细错误
node dist/main.js
```

**常见原因**：
- 缺少必要的环境变量
- 数据库连接失败
- 端口被占用

---

### 2. 前端无法连接后端

**症状**：浏览器显示 Network Error 或 502

**排查步骤**：
```bash
# 1. 确认后端正在运行
curl http://localhost:3001/api/v1/health

# 2. 检查 CORS 配置
grep -r "CORS" backend/src/

# 3. 检查前端 API 地址配置
cat frontend/.env | grep API
```

**解决方案**：
```javascript
// 后端 CORS 配置 (backend/src/main.ts)
app.enableCors({
  origin: process.env.FRONTEND_URL || 'http://localhost:3000',
  credentials: true,
});
```

---

### 3. 数据库连接失败

**错误信息**：
```
Error: P1001: Can't reach database server
```

**解决方案**：
```bash
# 检查 PostgreSQL 服务
sudo systemctl status postgresql

# 启动服务
sudo systemctl start postgresql

# 检查连接配置
psql -U postgres -h localhost -c "SELECT 1"
```

数据库配置示例：
```env
DATABASE_URL=postgresql://username:password@localhost:5432/auto_selection
```

---

### 4. Docker 容器启动失败

**错误信息**：
```
docker-compose up -d
# Error response from daemon: ...
```

**解决方案**：
```bash
# 查看详细日志
docker-compose logs

# 清理重建
docker-compose down -v
docker-compose build --no-cache
docker-compose up -d

# 检查 Docker 服务
sudo systemctl status docker
```

## 功能问题

### 1. 注册/登录失败

**症状**：提示"验证码错误"或"邮箱已被注册"

**排查步骤**：
1. 检查验证码是否过期（5分钟有效）
2. 确认邮箱是否已注册过
3. 检查网络连接

**解决方案**：
- 重新获取验证码
- 使用忘记密码功能重置
- 联系客服处理

---

### 2. AI 选品分析失败

**症状**：
- 提示"积分不足"
- 提示"服务繁忙"
- 一直显示"分析中"

**排查步骤**：
```bash
# 1. 检查积分余额
curl -H "Authorization: Bearer <token>" http://localhost:3001/api/v1/points/balance

# 2. 检查后端日志
pm2 logs backend --lines 50

# 3. 检查 AI 服务状态
curl http://localhost:3001/api/v1/health
```

**解决方案**：
- 充值积分或等待每日重置
- 刷新页面重试
- 重启后端服务

---

### 3. 翻译结果为空

**症状**：翻译后返回空白

**排查步骤**：
1. 检查输入文本是否为空
2. 确认语言选择正确
3. 检查积分是否充足

**解决方案**：
```bash
# 手动测试翻译接口
curl -X POST http://localhost:3001/api/v1/ai/translate \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer <token>" \
  -d '{"text":"测试","sourceLang":"zh","targetLang":"en"}'
```

---

### 4. 页面显示异常

**症状**：页面样式错乱或白屏

**排查步骤**：
1. 清除浏览器缓存
2. 尝试无痕/隐私模式
3. 检查控制台错误

**解决方案**：
```bash
# 重新构建前端
cd frontend
rm -rf .next
npm run build
```

## 性能问题

### 1. 服务响应缓慢

**排查步骤**：
```bash
# 检查 CPU 和内存使用
top
htop

# 检查磁盘 I/O
iostat -x 1 5

# 检查网络连接
netstat -an | grep ESTABLISHED | wc -l
```

**优化建议**：
- 增加服务器配置
- 启用 Redis 缓存
- 配置 Nginx 静态资源缓存

---

### 2. 数据库查询慢

**解决方案**：
```sql
-- 查看慢查询
SELECT * FROM pg_stat_statements 
ORDER BY mean_exec_time DESC 
LIMIT 10;

-- 添加索引
CREATE INDEX idx_user_email ON users(email);
CREATE INDEX idx_product_keyword ON products(keyword);
```

---

### 3. 内存泄漏

**症状**：服务运行一段时间后变慢或崩溃

**排查步骤**：
```bash
# 检查 PM2 内存使用
pm2 monit

# 查看详细内存
pm2 list
```

**解决方案**：
```bash
# 重启服务释放内存
pm2 restart backend

# 配置自动重启
pm2 start dist/main.js --name "backend" --max-memory-restart 500M
```

## 安全相关

### 1. JWT Token 过期

**症状**：突然需要重新登录

**排查步骤**：
```bash
# 检查 JWT 配置
grep JWT backend/.env

# 检查 Token 有效期配置
# JWT_EXPIRES_IN 默认 7d (7天)
```

**解决方案**：
```bash
# 生成新的 JWT Secret
node -e "console.log(require('crypto').randomBytes(64).toString('hex'))"

# 更新 .env 文件
JWT_SECRET=<new-secret>
JWT_EXPIRES_IN=7d
```

---

### 2. HTTPS 配置问题

**解决方案**：
```bash
# 安装 Certbot
sudo apt install -y certbot python3-certbot-nginx

# 获取证书
sudo certbot --nginx -d your-domain.com

# 自动续期
sudo certbot renew --dry-run
```

Nginx HTTPS 配置：
```nginx
server {
    listen 443 ssl http2;
    server_name your-domain.com;
    
    ssl_certificate /etc/letsencrypt/live/your-domain.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/your-domain.com/privkey.pem;
    
    location / {
        root /var/www/auto-selection/frontend;
        index index.html;
    }
    
    location /api {
        proxy_pass http://127.0.0.1:3001;
    }
}
```

## 服务管理

### 重启服务

#### PM2 重启
```bash
# 重启特定服务
pm2 restart backend

# 重启所有服务
pm2 restart all

# 优雅重载（零停机）
pm2 reload backend
```

#### Systemd 重启
```bash
# 重启服务
sudo systemctl restart auto-selection

# 查看状态
sudo systemctl status auto-selection

# 开机自启
sudo systemctl enable auto-selection
```

#### Docker 重启
```bash
# 重启所有容器
docker-compose restart

# 重启特定容器
docker-compose restart backend

# 强制重建
docker-compose down && docker-compose up -d
```

### 自动化健康检查

创建检查脚本 `/opt/check-health.sh`：

```bash
#!/bin/bash
# Health Check Script

API_URL="http://localhost:3001/api/v1/health"
FRONTEND_URL="http://localhost"

echo "=== Health Check $(date) ==="

# Check Backend
if curl -sf "$API_URL" > /dev/null; then
    echo "✅ Backend: OK"
else
    echo "❌ Backend: FAILED"
    pm2 restart backend
fi

# Check Frontend
if curl -sf "$FRONTEND_URL" > /dev/null; then
    echo "✅ Frontend: OK"
else
    echo "❌ Frontend: FAILED"
    sudo systemctl restart nginx
fi

echo "=== Check Complete ==="
```

添加定时任务：
```bash
# 编辑 crontab
sudo crontab -e

# 添加每小时检查一次
0 * * * * /opt/check-health.sh >> /var/log/health-check.log 2>&1
```
