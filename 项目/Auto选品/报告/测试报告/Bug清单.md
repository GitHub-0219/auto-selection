#### BUG-001：登录接口限流阈值未明确配置

| 属性 | 内容 |
|------|------|
| **Bug ID** | BUG-001 |
| **标题** | 登录接口暴力破解防护不足 |
| **优先级** | P1 |
| **严重程度** | 高 |
| **模块** | 用户模块 |
| **接口** | POST /auth/login |
| **发现时间** | 2026-04-09 |
| **状态** | 🟡 待修复 |

**问题描述**：
当前登录接口缺少明确的限流配置，存在被暴力破解的风险。

**复现步骤**：
1. 使用脚本对登录接口连续发送100次错误密码请求
2. 观察服务器响应

**预期行为**：
- 连续5次失败后应触发限流
- 返回429 Too Many Requests
- 15分钟后自动解除

**实际行为**：
- 未观察到明确的限流机制
- 所有请求均正常响应

**影响范围**：
- 所有用户账号安全

**修复建议**：
```typescript
// 实现登录限流守卫
@Injectable()
export class LoginThrottlerGuard implements CanActivate {
  private attempts = new Map<string, number>();
  private readonly MAX_ATTEMPTS = 5;
  private readonly LOCKOUT_DURATION = 15 * 60 * 1000; // 15分钟
  
  canActivate(context: ExecutionContext): boolean {
    const email = context.switchToHttp().getRequest().body.email;
    const attempts = this.attempts.get(email) || 0;
    
    if (attempts >= this.MAX_ATTEMPTS) {
      throw new ThrottlerException('登录尝试次数过多，请15分钟后重试');
    }
    
    return true;
  }
}
```

**责任人**：Backend Team  
**预计修复时间**：1天

### 🟡 P2 - 中优先级

#### BUG-003：邀请奖励重复邀请提示不区分状态

| 属性 | 内容 |
|------|------|
| **Bug ID** | BUG-003 |
| **标题** | 重复邀请未区分"已注册"和"已邀请"状态 |
| **优先级** | P2 |
| **严重程度** | 中 |
| **模块** | 邀请奖励模块 |
| **接口** | POST /invite/register |
| **发现时间** | 2026-04-09 |
| **状态** | 🟡 待修复 |

**问题描述**：
用户A使用邀请码注册后，用户B再次使用同一邀请码时提示"已邀请过"，但未区分是"被邀请人已注册"还是"被邀请人已领取奖励"。

**修复建议**：
增加更细化的状态码：
- `INVITE_ALREADY_USED`：邀请码已被使用
- `INVITE_SELF_USE`：不能邀请自己
- `INVITE_EXPIRED`：邀请码已过期
