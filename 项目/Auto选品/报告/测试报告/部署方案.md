## 一、服务器选型建议

### 1.1 推荐配置

| 环境 | 配置 | 适用场景 | 月费用估算 |
|------|------|----------|------------|
| **开发环境** | 1核2G | 开发测试 | ¥100-150 |
| **测试环境** | 2核4G | 功能/压力测试 | ¥200-300 |
| **预生产环境** | 2核4G | 上线前验证 | ¥200-300 |
| **生产环境** | 4核8G | MVP初期（500-2000用户） | ¥400-600 |

### 1.2 云服务商对比

| 云服务商 | 推荐配置 | 优点 | 缺点 |
|----------|----------|------|------|
| **阿里云ECS** | 2核4G 新人价 ¥99/月 | 中文文档、国内速度快 | 续费涨价 |
| **腾讯云CVM** | 2核4G ¥128/月 | 微信生态集成 | 文档稍乱 |
| **AWS EC2** | t3.medium | 全球CDN、易扩展 | 国内访问慢 |
| **火山引擎** | 2核4G ¥99/月 | 性价比高 | 生态较小 |

**推荐**：MVP阶段使用阿里云或火山引擎，后续根据用户量弹性扩展。

### 1.3 未来扩展路线

```
用户量增长 → 配置升级路径：

500-2000用户   →  4核8G  单机部署
2000-10000用户 →  4核8G  + Redis缓存
10000-50000用户 →  8核16G  + 数据库读写分离
50000+用户      →  Kubernetes容器编排
```

## 三、部署流程

### 3.1 环境准备

```bash
# 1. 服务器初始化
apt update && apt upgrade -y
apt install -y nginx certbot nodejs npm

# 2. 安装 Node.js 18+
curl -fsSL https://deb.nodesource.com/setup_18.x | bash -
apt install -y nodejs

# 3. 安装 PM2
npm install -g pm2

# 4. 安装 Docker (可选)
curl -fsSL https://get.docker.com | bash -
```

### 3.2 应用部署脚本

```bash
#!/bin/bash
# deploy.sh - 自动部署脚本

set -e

# 配置
APP_DIR="/var/www/auto-selection"
BACKEND_DIR="$APP_DIR/backend"
FRONTEND_DIR="$APP_DIR/frontend"
BACKUP_DIR="/var/backups/auto-selection"

# 颜色输出
GREEN='\033[0;32m'
YELLOW='\033[1;33m'
NC='\033[0m'

log() {
    echo -e "${GREEN}[INFO]${NC} $1"
}

warn() {
    echo -e "${YELLOW}[WARN]${NC} $1"
}

# 备份
backup() {
    log "开始备份..."
    timestamp=$(date +%Y%m%d_%H%M%S)
    mkdir -p $BACKUP_DIR
    
    # 备份数据库
    pg_dump -U postgres auto_selection > $BACKUP_DIR/db_$timestamp.sql
    
    # 备份配置
    cp -r $BACKEND_DIR/.env $BACKUP_DIR/.env_$timestamp
    
    log "备份完成: $BACKUP_DIR"
}

# 部署后端
deploy_backend() {
    log "部署后端服务..."
    cd $BACKEND_DIR
    
    # 拉取代码
    git pull origin main
    
    # 安装依赖
    npm ci
    
    # 构建
    npm run build
    
    # 重启服务
    pm2 restart backend --update-env
    pm2 save
    
    log "后端部署完成"
}

# 部署前端
deploy_frontend() {
    log "部署前端..."
    cd $FRONTEND_DIR
    
    # 拉取代码
    git pull origin main
    
    # 安装依赖
    npm ci
    
    # 构建
    npm run build
    
    # 同步到CDN/静态托管
    rsync -avz out/ user@cdn-server:/var/www/static/
    
    log "前端部署完成"
}

# 健康检查
health_check() {
    log "执行健康检查..."
    
    for i in {1..5}; do
        if curl -sf http://localhost:3001/api/health > /dev/null; then
            log "健康检查通过 ✓"
            return 0
        fi
        warn "等待服务启动... ($i/5)"
        sleep 5
    done
    
    warn "健康检查失败"
    return 1
}

# 主流程
main() {
    log "=== Auto选品部署开始 ==="
    
    backup
    deploy_backend
    deploy_frontend
    
    if health_check; then
        log "=== 部署成功 ==="
    else
        warn "=== 部署完成但健康检查未通过 ==="
    fi
}

main "$@"
```

### 3.3 Docker部署（可选）

```yaml
# docker-compose.yml
version: '3.8'

services:
  backend:
    build: ./backend
    ports:
      - "3001:3001"
    environment:
      - NODE_ENV=production
      - DATABASE_URL=postgresql://user:pass@db:5432/auto_selection
      - REDIS_URL=redis://cache:6379
    depends_on:
      - db
      - cache
    restart: always

  frontend:
    build: ./frontend
    ports:
      - "3000:3000"
    depends_on:
      - backend
    restart: always

  db:
    image: postgres:14
    volumes:
      - pgdata:/var/lib/postgresql/data
    environment:
      - POSTGRES_DB=auto_selection
      - POSTGRES_USER=user
      - POSTGRES_PASSWORD=pass

  cache:
    image: redis:7-alpine
    volumes:
      - redisdata:/data

volumes:
  pgdata:
  redisdata:
```

## 五、备份策略

### 5.1 备份计划

| 备份类型 | 频率 | 保留时间 | 存储位置 |
|----------|------|----------|----------|
| 全量备份 | 每日 02:00 | 30天 | 本地 + OSS |
| 增量备份 | 每6小时 | 7天 | 本地 |
| 数据库备份 | 每小时 | 24小时 | 本地 |
| 配置文件 | 每次部署前 | 90天 | OSS |
| 日志归档 | 每日 | 30天 | OSS |

### 5.2 备份脚本

```bash
#!/bin/bash
# backup.sh - 备份脚本

BACKUP_DIR="/var/backups/auto-selection"
DATE=$(date +%Y%m%d_%H%M%S)
DB_NAME="auto_selection"
DB_USER="postgres"

# 创建备份目录
mkdir -p $BACKUP_DIR/daily

# 数据库备份
echo "备份数据库..."
pg_dump -U $DB_USER $DB_NAME | gzip > $BACKUP_DIR/daily/db_${DATE}.sql.gz

# 配置文件备份
echo "备份配置文件..."
cp /var/www/auto-selection/backend/.env $BACKUP_DIR/config_${DATE}.env

# 清理旧备份（保留30天）
find $BACKUP_DIR/daily -mtime +30 -delete

# 上传到OSS（可选）
# ossutil cp -r $BACKUP_DIR oss://my-bucket/backups/

echo "备份完成: $DATE"
```
