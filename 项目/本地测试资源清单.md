| 端口 | 服务 | 说明 |


## 三、配置文件

### 1. 前端配置

**文件路径**: `./frontend/.env`

```env
NEXT_PUBLIC_API_URL=http://localhost:3001/api
```

### 2. 后端配置

**文件路径**: `./backend/.env`

```env
# Mock Server 配置
PORT=3001
NODE_ENV=development

# JWT 配置（Mock模式下可选）
JWT_SECRET=your-secret-key-here
JWT_EXPIRES_IN=7d

# 数据库配置（Mock模式下可选）
DATABASE_URL=postgresql://user:password@localhost:5432/autoselection
```

### 3. API 基础地址

- **开发环境**: `http://localhost:3001/api`
- **生产环境**: `https://api.autoselection.com/api`

## 七、快速启动

### 使用一键脚本 (Linux/macOS)

```bash
# 创建日志目录
mkdir -p logs

# 添加执行权限
chmod +x start-local.sh

# 运行部署脚本
./start-local.sh
```

### 手动启动

```bash
# 1. 启动后端 Mock 服务
cd backend
npm install
npm run mock

# 2. 新开终端，启动前端
cd frontend
npm install
npm run dev
```

### Windows 用户

```bash
# 使用 PowerShell 或 CMD
# 启动后端
cd backend
npm install
npm run mock

# 新开窗口，启动前端
cd frontend
npm install
npm run dev
```

## 九、常见问题

### Q1: 前端无法连接后端

**原因**: 后端服务未启动或端口被占用

**解决**:
```bash
# 检查端口占用
lsof -i :3001

# 重启后端服务
cd backend
npm run mock
```

### Q2: 依赖安装失败

**原因**: 网络问题或 Node.js 版本过低

**解决**:
```bash
# 清理缓存
npm cache clean --force

# 使用淘宝镜像
npm config set registry https://registry.npmmirror.com
npm install
```

### Q3: 端口被占用

**解决**:
```bash
# 查找占用进程 (Linux/macOS)
lsof -i :3000
kill -9 <PID>

# Windows
netstat -ano | findstr :3000
taskkill /PID <PID> /F
```
