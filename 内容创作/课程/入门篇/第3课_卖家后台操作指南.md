### 【第一部分】后台首页与核心模块概览

#### 1.1 登录与界面布局

**登录方式：**

方式一：电脑端
1. 打开 TikTok Shop Seller Center（seller-xxxx.tiktok.com，xxxx为目标市场缩写）

3. 输入密码登录

1. 下载 TikTok Shop Seller App
2. 打开App，输入账号密码
3. 完成二次验证

### 【第二部分】商品管理模块详解

#### 2.1 新增商品流程

上架商品是后台最核心的操作之一。以下是完整的步骤：

**第一步：进入新增商品页面**

路径：商品 → 新增商品

**第二步：填写商品基本信息**

| 字段 | 填写要求 | 注意事项 |
|------|----------|----------|
| 商品名称 | 60字符以内 | 包含核心关键词，利于搜索 |
| 商品描述 | 500字符以内 | 突出卖点，使用场景 |
| 商品类目 | 三级类目 | 选择最精准的类目 |
| 品牌 | 如有品牌需填写 | 无品牌填"无"或"Other" |

**商品名称规范：**
```
格式建议：[核心词] + [属性] + [特点/卖点]

示例：
✅ 好用多功能切菜器 不锈钢厨房必备
✅ 2024新款防晒喷雾 SPF50+ 防水防汗
❌ 切菜器
❌ 超级好用的那个切菜的东西
```

**第三步：设置SKU（库存单位）**

SKU是指商品的具体规格，如颜色、尺寸等。

| 操作 | 说明 |
|------|------|
| 添加SKU属性 | 如颜色、尺寸、规格 |
| 设置SKU编码 | 方便库存管理，建议自行编写 |
| 设置SKU图片 | 不同规格上传对应图片 |
| 设置库存数量 | 填写各SKU的库存 |
| 设置SKU价格 | 填写各SKU的售价 |

**SKU设置注意事项：**
- SKU属性不超过3个维度
- 每个SKU必须有独立库存
- 颜色/尺寸要准确，不能有歧义
- 建议设置"买家下单前必选"而非"买家可选可不选"

**第四步：上传商品图片和视频**

| 素材类型 | 要求 | 建议 |
|----------|------|------|
| 主图（封面） | 800x800px以上，JPG/PNG | 白色背景，纯商品展示 |
| 副图 | 800x800px以上，最多9张 | 展示细节、使用场景、对比 |
| 视频 | 建议上传，9:16竖版 | 展示商品功能，建议15-30秒 |

**图片规范：**
```
主图要求：
✅ 纯白底或浅色背景
✅ 商品占图片80%以上
✅ 无水印、无文字
✅ 清晰无模糊

副图建议：
✅ 第二张：商品细节图
✅ 第三张：使用场景图
✅ 第四张：尺寸参照图
✅ 第五张：对比图（如有）
```

**第五步：设置物流信息**

| 设置项 | 说明 |
|--------|------|
| 发货方式 | 自发货运送 / TikTok发货 |
| 运费模板 | 选择已创建的运费模板 |
| 发货时间 | 设置发货天数（1-7天） |
| 退货政策 | 设置是否支持退货 |

**发货时间设置建议：**
- 新手建议设置为3天，给自己留出备货时间
- 有稳定供应链后可设置为1-2天
- 不要设置7天以上，会影响转化率

**第六步：提交审核**

点击"提交"后，商品会进入平台审核。

| 审核状态 | 说明 |
|----------|------|
| 审核中 | 等待平台审核，通常1-24小时 |
| 已上线 | 审核通过，商品已展示 |
| 审核拒绝 | 未通过审核，查看原因修改后重新提交 |

#### 2.2 商品列表与批量管理

**商品列表页面功能：**

| 功能 | 说明 |
|------|------|
| 搜索商品 | 支持按商品名称、SKU、ID搜索 |
| 筛选商品 | 按状态、类目、价格区间筛选 |
| 批量操作 | 批量上下架、批量编辑、批量删除 |
| 排序 | 按创建时间、销量、价格排序 |
| 导出数据 | 导出商品列表Excel |

**批量操作示例：**

场景：想把20个商品同时下架

操作步骤：
1. 进入"商品列表"页面
2. 勾选需要下架的商品（可使用筛选+全选）
3. 点击"批量操作"
4. 选择"批量下架"
5. 确认操作

**商品状态说明：**

| 状态 | 说明 | 买家端展示 |
|------|------|------------|
| 上架中 | 正常在售 | 可见 |
| 已下架 | 手动下架 | 不可见 |
| 审核拒绝 | 未通过审核 | 不可见 |
| 缺货 | 库存为0 | 不可见（部分市场可显示） |
| 售完 | 库存消耗完 | 可能显示"到货通知" |

#### 2.3 商品分组管理

**为什么需要商品分组？**

商品分组可以帮助你更好地管理商品，方便：
- 快速找到某类商品
- 对某类商品统一设置活动
- 制作营销素材时快速选取

**如何创建商品分组：**

路径：商品 → 商品分组 → 新增分组

| 步骤 | 操作 |
|------|------|
| 1 | 点击"新增分组" |
| 2 | 输入分组名称（如"引流款"、"利润款"） |
| 3 | 选择分组内的商品（可手动选择或按条件筛选） |
| 4 | 保存分组 |

### 【第三部分】订单管理模块详解

#### 3.1 订单状态流转

TikTok Shop的订单从创建到完成，经历以下状态：

```
订单创建 → 待付款 → 已付款 → 待发货 → 运输中 → 已签收 → 已完成
                  ↓                                    ↓
              已取消                              退款/售后
                  ↓                                    ↓
              退款中                               退款完成
```

**各状态详细说明：**

| 状态 | 说明 | 超时要求 |
|------|------|----------|
| 待付款 | 买家已下单，但未付款 | 买家有30分钟-24小时付款时间 |
| 已付款 | 买家已付款，等待发货 | 必须48小时内发货 |
| 待发货 | 等待商家发货 | 超时将受到处罚 |
| 运输中 | 物流已揽收，运输中 | 等待买家签收 |
| 已签收 | 买家已签收 | 等待买家确认或自动确认 |
| 已完成 | 订单完成 | 买家确认或超时自动确认 |
| 已取消 | 订单取消 | 可能涉及退款 |
| 退款/售后 | 买家申请退款或售后 | 需要商家处理 |

#### 3.2 待发货订单处理

**待发货是日常运营中最需要关注的订单状态。**

| 操作 | 说明 |
|------|------|
| 查看详情 | 点击订单查看买家信息、商品信息、收货地址 |
| 标记发货 | 选择物流渠道，填写物流单号 |
| 取消订单 | 特殊情况下可取消订单（如缺货） |
| 联系买家 | 有问题可联系买家沟通 |

**发货流程详解：**

**步骤一：打印面单**

路径：订单 → 待发货 → 勾选订单 → 打印面单

| 面单类型 | 说明 |
|----------|------|
| 纸质面单 | 打印后贴在包裹上 |
| 电子面单 | 平台对接物流公司，直接获取面单号 |

**步骤二：打包发货**

| 操作 | 说明 |
|------|------|
| 打包商品 | 按订单商品打包 |
| 贴面单 | 面单贴在包裹外侧 |
| 交付物流 | 交给快递揽收 |

**步骤三：填写物流信息**

路径：订单 → 待发货 → 点击"确认发货" → 填写物流信息

| 信息项 | 填写说明 |
|--------|----------|
| 物流渠道 | 选择使用的物流公司 |
| 物流单号 | 填写物流单号 |

**步骤四：提交发货**

确认信息无误后，点击"确认发货"。

**发货时效要求：**

| 市场 | 发货时效要求 | 超时处罚 |
|------|-------------|----------|
| 马来西亚 | 48小时内 | 延迟发货率>10%将受到处罚 |
| 泰国 | 48小时内 | 延迟发货率>10%将受到处罚 |
| 越南 | 48小时内 | 延迟发货率>10%将受到处罚 |
| 菲律宾 | 48小时内 | 延迟发货率>10%将受到处罚 |
| 印尼 | 48小时内 | 延迟发货率>10%将受到处罚 |

**⚠️ 重要提醒：48小时发货时效是从买家付款时间开始计算，不是下单时间！**

#### 3.3 退款/售后处理

**退款类型：**

| 类型 | 说明 | 处理时限 |
|------|------|----------|
| 仅退款 | 买家未收到货，申请退款 | 48小时内处理 |
| 退货退款 | 买家已收到货，退货退款 | 48小时内处理 |
| 退货 | 买家申请退货，商家寄回商品 | 按退货协议处理 |

**处理流程：**

| 步骤 | 操作 | 说明 |
|------|------|------|
| 1 | 查看退款申请 | 了解退款原因和买家诉求 |
| 2 | 联系买家（可选） | 尝试解决问题，避免退款 |
| 3 | 审核退款申请 | 同意或拒绝退款 |
| 4 | 执行退款 | 退款原路返回买家账户 |

**处理建议：**

| 情况 | 建议操作 |
|------|----------|
| 商品已发货，买家未收到 | 联系物流确认状态，必要时补发 |
| 商品有瑕疵/错误 | 直接同意退款，避免差评 |
| 买家误操作 | 联系买家说明情况，引导取消退款 |
| 无理由退款 | 评估成本后决定，差评风险高可同意 |

### 【第四部分】数据与财务模块

#### 4.1 核心数据看板

路径：数据 → 核心数据

**核心指标解读：**

| 指标 | 定义 | 分析价值 |
|------|------|----------|
| GMV | 总成交金额 | 衡量店铺规模 |
| 订单数 | 成交订单数量 | 衡量订单量 |
| 客单价 | GMV ÷ 订单数 | 反映客户购买力 |
| 转化率 | 访客→买家转化 | 反映商品吸引力 |
| 访客数 | 进店人数 | 反映流量规模 |
| 退款率 | 退款订单÷总订单 | 反映商品质量和服务 |
| 好评率 | 好评数÷总评价数 | 反映用户满意度 |

**数据时间维度：**

| 维度 | 说明 | 适用场景 |
|------|------|----------|
| 今日 | 当天实时数据 | 关注日常运营 |
| 近7天 | 最近7天累计 | 周度复盘 |
| 近30天 | 最近30天累计 | 月度分析 |
| 自定义 | 自选时间段 | 特定活动分析 |

#### 4.2 商品数据分析

路径：数据 → 商品分析

**单品数据指标：**

| 指标 | 说明 | 优化方向 |
|------|------|----------|
| 商品访客数 | 看到商品详情页的人数 | 优化主图、标题 |
| 商品加购率 | 加购人数÷访客数 | 优化详情页 |
| 商品转化率 | 下单人数÷访客数 | 优化价格、主图 |
| 商品销售额 | 单品成交金额 | 综合优化 |
| 商品评价 | 买家评价内容 | 提升产品质量和服务 |

**数据导出：**

支持导出Excel，方便深度分析。

#### 4.3 财务账单管理

路径：资产 → 账单中心

**账单类型：**

| 类型 | 说明 |
|------|------|
| 收入 | 买家付款金额 |
| 支出 | 平台佣金、物流费、退款金额 |
| 结算 | 实际到账金额 |

**结算周期：**

| 店铺类型 | 结算周期 | 说明 |
|----------|----------|------|
| 跨境店 | 15-30天 | 订单完成后，等待账期结算 |
| 本土店 | 7-15天 | 账期相对较短 |

**提现规则：**

| 项目 | 说明 |
|------|------|
| 最低提现金额 | $10 或等值当地货币 |
| 提现方式 | 绑定Payoneer/PingPong等账户 |
| 提现时间 | 申请后1-3个工作日到账 |
| 手续费 | 按平台规则，部分由平台承担 |

### 【第五部分】实操演示

#### 5.1 上架一个商品（完整演示）

**演示商品：** 多功能切菜器

**步骤：**
1. 进入商品 → 新增商品
2. 填写商品名称："多功能切菜器 厨房用品 不锈钢"
3. 选择类目：家居用品 > 厨房用品 > 厨房工具
4. 填写商品描述（突出卖点、使用场景）
5. 设置SKU（颜色：银色/黑色，规格：单件/套装）
6. 上传主图和副图
7. 上传视频（15秒功能演示）
8. 设置价格（银色单件$8.99，黑色单件$8.99，套装$15.99）
9. 设置库存（各50件）
10. 选择运费模板
11. 设置发货时间（3天）
12. 提交审核

#### 5.2 处理一个订单（完整演示）

**场景：** 收到一个待发货订单，需要发货

**步骤：**
1. 进入订单 → 待发货

## 三、核心案例

### 案例：因不熟悉后台而踩坑的教训

**背景：**
卖家小张，第一次做电商，完全没有后台操作经验，2024年4月开始做TikTok Shop马来西亚跨境店。

**踩坑一：价格设置错误**

上架一款化妆刷时，小张把"12.99"填成了"1.299"，以为是系统会自动换算（把林吉特当人民币）。

结果：3个小时内被下了80多单，亏了将近1,000元。

教训：设置价格时一定要仔细核对货币单位和数值！

**踩坑二：发货超时被处罚**

第一批订单来了，小张很开心，但不知道怎么打印面单、怎么发货，拖了3天才发货。

结果：因发货超时被平台扣分，店铺评分下降。

教训：一定要提前熟悉发货流程，设置合理的发货时间！

**踩坑三：不知道怎么看数据**

小张每天就看有没有订单，从来不看后台数据面板。

结果：一个月后才发现某个商品访客很多但转化率极低，错过了优化时机。

教训：养成每天看数据的习惯，数据会告诉你问题在哪里！

**踩坑四：退货处理不当**

有个买家申请退货，理由是"商品与描述不符"。小张不知道怎么处理，就放着不管。

结果：48小时后系统自动同意退款，而且因为处理超时被扣分。

教训：一定要及时处理退款申请，不懂就问客服！

**最终总结：**

小张后来花了一周时间专门学习后台操作，才慢慢步入正轨。

**给新手的忠告：**
1. 开店前先熟悉后台，不要急着上品
2. 上品前先测试一遍完整流程
3. 每天看数据，每周做复盘
4. 有问题及时问客服
